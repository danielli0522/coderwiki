# Story 2.3: Repository Management - Repository Deletion

## 基本信息

**Story ID:** 2.3  
**Epic:** Epic-2 (Repository Management)  
**Story 名称:** Repository Management - Repository Deletion  
**优先级:** 中  
**估计工时:** 10小时  
**状态:** Ready for Review  
**负责人:** 全栈开发人员  

## 用户故事

**作为** 系统用户，  
**我想要** 删除不再需要的仓库，  
**以便** 清理系统空间并移除相关的文档和数据。

## 验收标准

- [ ] 用户能够通过确认对话框删除仓库
- [ ] 系统能够级联删除仓库相关的所有文档
- [ ] 系统能够级联删除仓库相关的所有任务
- [ ] 系统能够清理本地存储的仓库文件
- [ ] 删除操作后界面能够实时更新
- [ ] 系统能够处理删除失败的情况并提供友好的错误提示
- [ ] 删除操作需要有适当的权限验证

## Dev Notes

### Previous Story Insights
根据Story 2.1，系统已经实现了仓库添加功能，数据库结构包含仓库表(repositories)和相关的关系表。仓库模型与文档表(documents)和任务表(tasks)存在级联关系。删除仓库时需要处理这些关联数据的清理。

```sql
-- 已有的仓库表结构
CREATE TABLE repositories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    name VARCHAR(255) NOT NULL,
    url VARCHAR(500) NOT NULL,
    description TEXT,
    status ENUM('pending', 'analyzing', 'completed', 'failed') DEFAULT 'pending',
    last_analysis DATETIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    UNIQUE KEY uk_user_url (user_id, url)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### Data Models
基于现有数据库结构，删除操作需要处理以下关联表：

1. **文档表(Document)**: 包含repository_id外键，设置了ON DELETE CASCADE
2. **任务表(Task)**: 包含repository_id外键，设置了ON DELETE CASCADE
3. **本地文件系统**: 需要手动清理克隆的仓库文件

### API Specifications
需要实现以下API接口：

1. **DELETE /api/repositories/{id}** - 删除仓库
   - 请求头：需要认证token
   - 响应：{"success": true, "message": "Repository deleted successfully"}

2. **GET /api/repositories/{id}/confirm-delete** - 获取删除确认信息
   - 响应：{"repository": {...}, "documents_count": 5, "tasks_count": 3}

### Component Specifications
前端组件需要包含：

1. **Delete Confirmation Modal** - 删除确认对话框
   - 显示仓库基本信息
   - 显示关联文档和任务数量
   - 确认和取消按钮

2. **Repository Actions Component** - 仓库操作组件
   - 删除按钮
   - 状态检查

3. **Cleanup Status Component** - 清理状态显示
   - 删除进度显示
   - 错误信息展示
   - 成功提示

## 风险和注意事项

### 技术风险
- **数据丢失:** 删除操作不可逆，可能导致重要数据丢失
- **文件系统权限:** 可能因权限问题导致文件删除失败
- **并发删除:** 多用户同时删除可能导致数据不一致
- **大文件删除:** 大仓库删除可能耗时较长

### 解决方案
- 实现多级确认机制
- 添加详细的权限检查和错误处理
- 使用数据库锁机制保证并发安全
- 实现异步删除和进度显示

### 业务风险
- **用户体验:** 删除操作可能影响用户体验
- **数据完整性:** 删除操作可能影响数据完整性
- **系统性能:** 大量删除操作可能影响系统性能

### 解决方案
- 提供清晰的删除确认信息
- 实现事务处理保证数据一致性
- 实现队列和限流机制
