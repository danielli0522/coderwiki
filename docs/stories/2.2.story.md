# Story 2.2: Repository Management - Repository Listing

## 基本信息

**Story ID:** 2.2  
**Epic:** Epic-2 (Repository Management)  
**Story 名称:** Repository Management - Repository Listing  
**优先级:** 高  
**估计工时:** 16小时  
**状态:** Ready for Review  
**负责人:** 全栈开发人员  

## 用户故事

**作为** 系统用户，  
**我想要** 在仪表板中查看我所有仓库的列表和状态，  
**以便** 能够快速了解仓库状态并进行管理操作。

## 验收标准

- [ ] 用户能够在主仪表板中查看所有已添加的仓库列表
- [ ] 每个仓库显示基本信息（名称、描述、URL、状态）
- [ ] 仓库状态指示器能够正确显示（pending、analyzing、completed、failed）
- [ ] 仓库列表支持分页和排序功能
- [ ] 用户能够对仓库进行基本管理操作（查看详情、删除、重新分析）
- [ ] 仓库状态变化时界面能够实时更新
- [ ] 界面在不同设备上（桌面、平板、手机）都能良好显示
- [ ] 仓库列表能够响应式地适应不同屏幕尺寸

### Data Models Extension
基于已设计的数据库结构，需要扩展以下数据模型：

```python
# backend/app/models/repository.py (扩展)
class Repository(db.Model):
    __tablename__ = 'repositories'
    
    # 现有字段...
    id = db.Column(db.Integer, primary_key=True)
    user_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=False)
    name = db.Column(db.String(255), nullable=False)
    url = db.Column(db.String(500), nullable=False, unique=True)
    description = db.Column(db.Text)
    status = db.Column(db.String(50), default='pending')
    last_analysis = db.Column(db.DateTime)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    # 新增字段用于列表显示
    language = db.Column(db.String(100))  # 主要编程语言
    star_count = db.Column(db.Integer, default=0)  # 星标数量
    fork_count = db.Column(db.Integer, default=0)  # 分支数量
    last_commit = db.Column(db.String(50))  # 最后提交SHA
    commit_count = db.Column(db.Integer, default=0)  # 提交次数
    repo_size = db.Column(db.Integer, default=0)  # 仓库大小(KB)
    
    # 关系
    user = db.relationship('User', backref='repositories')
    documents = db.relationship('Document', backref='repository', lazy=True)
    tasks = db.relationship('Task', backref='repository', lazy=True)
```

### API Specifications
需要实现以下API接口：

1. **GET /api/repositories** - 获取用户仓库列表（分页）
   - 参数：page=1, per_page=10, sort_field=created_at, sort_order=desc
   - 响应：{"repositories": [...], "total": 20, "page": 1, "per_page": 10}

2. **GET /api/repositories/{id}/status** - 获取仓库状态
   - 响应：{"id": 1, "status": "analyzing", "progress": 65}

3. **POST /api/repositories/{id}/reanalyze** - 重新分析仓库
   - 响应：{"success": true, "message": "Reanalysis started"}

4. **GET /api/repositories/stats** - 获取仓库统计信息
   - 响应：{"total": 20, "completed": 15, "analyzing": 3, "failed": 2}
