## Dev Notes

### Previous Story Insights
Story 1.3已经实现了用户注册功能，包括用户模型、认证服务和注册API。本故事将在此基础上实现用户登录和退出功能。

### Data Models
复用Story 1.3中创建的User模型：
- id: 主键
- username: 用户名
- email: 邮箱
- password_hash: 加密密码
- created_at: 创建时间
- updated_at: 更新时间
- is_active: 是否激活

[Source: docs/architecture/data-models.md]

### API Specifications
登录API端点：
- POST /api/auth/login
- POST /api/auth/logout
- GET /api/auth/status

登录请求体：
```json
{
    "username": "string",
    "password": "string"
}
```

登录成功响应：
```json
{
    "success": true,
    "user": {
        "id": 1,
        "username": "string",
        "email": "string"
    }
}
```

登录状态响应：
```json
{
    "logged_in": true,
    "user": {
        "id": 1,
        "username": "string",
        "email": "string"
    }
}
```

[Source: docs/architecture/rest-api-spec.md]

## Project Structure Notes

用户登录功能的实现建立在用户注册功能的基础上：

1. **数据层**: 复用User模型进行用户数据管理
2. **服务层**: 扩展AuthService添加登录和登出逻辑
3. **API层**: 扩展认证API添加登录、登出和状态检查
4. **会话层**: 使用Flask-Login进行会话管理
5. **视图层**: 登录页面负责用户界面
6. **交互层**: JavaScript负责用户交互和状态管理
