## 二、核心职责总结 (Core Responsibilities Summary)

[ **重要提示**：以下分类是您最终输出报告的结构基础。您必须严格按照这些分类来组织 API。]

- **提供者职责 (Providers):**
  - **数据包生命周期管理:** 提供完整的数据包导入导出流程管理。
  - **用户身份管理:** 支持多租户用户身份切换和权限控制。
  - **服务代理:** 作为中间层代理 DAP 和 DMP 服务的请求。
  - **RDC 集成:** 接收和处理 RDC 推送的数据包。
  - **文件存储:** 提供文件上传和 OSS 签名服务。
  - **状态监控:** 提供导入导出任务的状态查询和日志记录。
- **调用者职责 (Consumers):**
  - **DMP 系统集成:** 调用数见系统的报表解析、导出、导入等接口。
  - **DAP 系统集成:** 调用数芯系统的数据服务、导入导出等接口。
  - **OSS 存储服务:** 调用对象存储服务进行文件管理。
  - **缓存服务:** 使用 Redis 进行数据缓存和会话管理。
  - **消息队列:** 使用消息队列进行异步任务处理。
  - **第三方认证:** 集成第三方系统的认证和授权服务。
