# CoderWiki业务领域模型

## 核心实体

### Repository (代码仓库)
- **概念**: 用户管理的代码项目，是系统的核心业务对象
- **模型位置**: `backend/app/models/repository.py`
- **关键属性**:
  - `name`: 仓库名称
  - `url`: Git仓库URL或本地路径
  - `source_type`: 来源类型 (git_remote, local_directory)
  - `analysis_status`: 分析状态 (pending, running, completed, failed)
  - `user_id`: 所属用户ID
  - `local_source_path`: 本地路径(仅本地仓库)

- **生命周期**: 导入 → 分析 → 文档生成 → 维护更新
- **业务规则**:
  - 每个用户可管理多个仓库
  - 仓库必须先完成分析才能生成文档
  - 支持Git远程仓库和本地目录两种来源
  - 本地路径在同用户内必须唯一

### User (用户)
- **概念**: 系统使用者，拥有和管理代码仓库
- **模型位置**: `backend/app/models/user.py`
- **关键属性**:
  - `username`: 用户名
  - `email`: 邮箱
  - `password_hash`: 密码哈希
  - `repositories`: 关联的仓库列表

- **默认账户** (开发环境):
  - 管理员：admin/admin123
  - 演示用户：demo/demo123
  - 测试用户：testuser/test123

### Analysis (分析任务)
- **概念**: 对代码仓库的结构化分析过程
- **状态机**: pending → running → completed → failed
- **分析内容**:
  - 代码文件结构和组织
  - 编程语言和框架识别
  - 依赖关系分析
  - 代码质量和复杂度指标
- **触发条件**: 手动触发或仓库更新时自动触发
- **输出位置**: 结构化分析报告存储在数据库

### Document (生成文档)
- **概念**: 基于分析结果生成的智能文档
- **文档类型**:
  - API文档：接口说明和使用示例
  - 架构文档：系统设计和组件关系
  - 使用指南：部署和开发说明
- **生成策略**: 基于代码结构分析 + AI推理
- **存储位置**: `coderwiki-output-docs/repos/{repo_name}/`

## 业务流程

### 仓库导入流程
1. **信息收集**: 用户提供仓库URL或本地路径
2. **验证检查**: 系统验证仓库有效性和访问权限
3. **实体创建**: 创建Repository实体并设置初始状态
4. **分析初始化**: 创建Analysis任务并加入处理队列
5. **状态返回**: 返回仓库ID和当前状态给用户

### 代码分析流程
1. **仓库克隆/访问**: 获取代码到本地分析环境
2. **结构扫描**: Analysis Engine扫描文件结构
3. **技术识别**: 提取编程语言、框架、依赖信息
4. **质量分析**: 计算代码复杂度和质量指标
5. **报告生成**: 生成结构化的分析报告
6. **状态更新**: 更新Repository的analysis_status

### 文档生成流程
1. **分析读取**: 读取完成的代码分析报告
2. **类型确定**: 基于项目特点确定文档类型和结构
3. **AI生成**: 调用AI服务生成各类型文档内容
4. **模板应用**: 应用项目特定的模板和样式
5. **文件输出**: 输出到`coderwiki-output-docs`目录
6. **索引更新**: 更新文档索引便于查找

### 用户权限流程
1. **身份验证**: 基于session的用户认证
2. **权限检查**: 用户只能访问自己的仓库
3. **操作授权**: 仓库的创建、分析、删除权限控制

## 业务规则

### 仓库管理规则
- 用户只能管理自己创建的仓库
- 仓库名称在用户范围内必须唯一
- 本地仓库路径在用户范围内必须唯一
- 删除仓库会同时删除相关的分析和文档数据

### 分析任务规则
- 同一仓库同时只能有一个运行中的分析任务
- 分析失败后可以重试，不限制重试次数
- 分析结果会覆盖之前的分析数据

### 文档生成规则
- 只有分析完成的仓库才能生成文档
- 文档生成失败不影响分析结果
- 重新生成文档会覆盖之前的文档文件

## 扩展点

### 支持的仓库类型
- **当前支持**: Git远程仓库、本地目录
- **计划支持**: GitLab私有仓库、SVN仓库、压缩包导入

### 支持的编程语言
- **当前支持**: Python、JavaScript、Java、Go等主流语言
- **扩展机制**: 语言解析器插件化设计

### AI服务集成
- **当前支持**: OpenAI GPT-4、Anthropic Claude
- **扩展机制**: 统一的LLM客户端接口
