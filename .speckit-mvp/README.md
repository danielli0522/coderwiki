# SpecKit MVP - Knowledge Base集成方案

**版本**: MVP 1.0
**创建日期**: 2025-09-28
**适用项目**: CoderWiki

## 🎯 MVP目标

验证通过结构化知识库(knowledge base)提升AI编程效率的核心假设：
- AI理解项目架构的时间从5-10分钟降低到30秒内
- AI能准确推荐项目特定的代码模式和最佳实践
- 新功能开发时AI建议符合项目规范和架构原则

## 📁 目录结构

```
.speckit-mvp/
├── knowledge/                    # 项目知识库
│   ├── architecture/
│   │   └── system-overview.md   # 系统架构概览
│   ├── business/
│   │   └── domain-model.md      # 业务领域模型
│   └── patterns/
│       └── common-patterns.md   # 常用代码模式
├── scripts/
│   └── knowledge-sync.sh        # 知识同步脚本
└── README.md                    # 本文档
```

## 🚀 快速开始

### 1. 执行知识同步
```bash
# 将knowledge内容同步到CLAUDE.md
./.speckit-mvp/scripts/knowledge-sync.sh

# 检查同步结果
head -50 CLAUDE.md
```

### 2. 验证AI效果

立即测试以下3个核心场景：

#### 场景1: 架构理解测试
询问AI: "请介绍CoderWiki的系统架构"
**期望**: AI能快速准确描述系统组件、分层结构、技术栈

#### 场景2: 代码模式推荐测试
询问AI: "我要新增一个用户管理的API端点，应该怎么做？"
**期望**: AI推荐标准的API模式，包括认证、验证、错误处理

#### 场景3: 新功能开发测试
询问AI: "帮我设计一个代码评分功能的数据模型"
**期望**: AI基于现有Repository模式设计，遵循项目命名和结构约定

## 📊 验证标准

### 成功指标
- ✅ AI能在1分钟内准确回答架构问题
- ✅ AI推荐的代码模式符合项目规范
- ✅ AI建议的实现路径具体可行
- ✅ AI使用的文件路径和模块名称准确

### 失败指标
- ❌ AI仍需要扫描代码才能回答问题
- ❌ AI推荐的模式与项目风格不符
- ❌ AI给出过于通用的建议，缺乏项目特异性
- ❌ AI提及不存在的文件或模块

## 🔄 使用工作流

### 日常使用
1. **开发新功能前**: 运行sync脚本确保knowledge最新
2. **询问AI建议**: 直接询问架构、模式、实现建议
3. **验证建议**: 对比AI建议与现有代码的一致性

### 知识维护
1. **重要架构变更后**: 更新相应的knowledge文档
2. **新模式确立后**: 添加到common-patterns.md
3. **定期同步**: 执行sync脚本更新CLAUDE.md

## 📝 Knowledge文档说明

### system-overview.md
- **内容**: 系统整体架构、组件职责、技术栈、设计决策
- **维护频率**: 重大架构变更时更新
- **使用场景**: AI需要理解项目整体结构时

### domain-model.md
- **内容**: 核心业务实体、业务流程、规则约束
- **维护频率**: 新增业务功能时更新
- **使用场景**: AI需要理解业务逻辑和数据模型时

### common-patterns.md
- **内容**: API、服务、模型、测试的标准代码模式
- **维护频率**: 确立新编码模式时更新
- **使用场景**: AI推荐具体代码实现时

## ⚠️ 注意事项

### 文件管理
- sync脚本会备份原CLAUDE.md为`.backup`文件
- 手动修改CLAUDE.md可能被sync脚本覆盖
- knowledge文档是单一数据源，优先维护它们

### 性能考虑
- MVP版本knowledge内容较少，同步速度快
- 未来扩展时需要考虑内容压缩和分层加载
- 避免knowledge文档过于冗长影响AI理解

### 维护成本
- 确保knowledge文档与代码实际情况一致
- 定期review文档准确性
- 及时更新过时信息
