### 🔄 BMAD-Method 工作流程

BMAD 文档生成器将按照以下 5 个阶段执行：

1. **代码分析阶段**

   - 扫描整个代码库
   - 识别技术模式和框架
   - 分析代码组织和架构
   - 评估代码质量和可维护性

2. **架构分析阶段**

   - 分析当前系统架构
   - 评估技术选择和模式
   - 设计最优架构方案
   - 评估可扩展性和性能

3. **流程分析阶段**

   - 分析数据流和过程模式
   - 识别工作流瓶颈
   - 映射集成点和数据依赖
   - 优化过程流和工作流

4. **问题诊断阶段**

   - 识别和分析技术问题
   - 进行根本原因分析
   - 制定综合解决方案
   - 评估风险和缓解策略

5. **文档生成阶段**
   - 规划和构建技术文档
   - 创建全面、组织良好的内容
   - 确保文档质量和完整性
   - 组织信息以获得最佳可读性

### 💡 使用建议

1. **完整文档生成**: 使用 BMAD 文档团队一次性生成所有文档
2. **分步骤生成**: 使用特定子代理逐步生成不同部分的文档
3. **自定义需求**: 在调用时指定特定的文档需求和要求
4. **质量保证**: 每个子代理都会确保文档的专业性和完整性
