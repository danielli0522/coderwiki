技术选型与对比：为什么是浏览器端？

GitNexus 的每一个关键选型背后的思考

为什么选择"零服务器"？

大部分代码分析工具都需要服务器：把代码上传到云端解析，返回结果。GitNexus 反其道而行——全部在浏览器或本机完成。为什么？

🔒

企业代码不出墙

企业客户的源代码是核心资产。要求上传到第三方服务器？门都没有。GitNexus 的零服务器架构从根上消除了这个障碍。

⚡

零延迟体验

本地解析 + 本地查询 = 毫秒级响应。不需要等网络传输、不需要排队等服务器资源。WebGL 渲染数千节点丝滑流畅。

💰

零运维成本

不需要维护服务器集群、不需要扩缩容、不需要担心宕机。开源项目也没有服务器账单的压力。

关键技术选型

TS
Tree-sitter 而非 LSP

LSP 虽然语义更丰富，但需要为每种语言启动独立的语言服务器进程，不适合浏览器环境。Tree-sitter 是轻量级的 AST 解析器，能编译成 WASM，14 种语言共享同一套处理逻辑。

DB
LadybugDB 而非 Neo4j

Neo4j 是重量级图数据库，需要 Java 运行时和独立服务器进程。LadybugDB（原 KuzuDB）是嵌入式图数据库，支持向量搜索，能编译成 WASM 在浏览器里跑。原生模式和 WASM 模式共享同一套查询接口。

EM
transformers.js 而非 OpenAI API

用 HuggingFace 的 transformers.js 在本地生成嵌入向量（Snowflake arctic-embed-xs, 384维），不依赖任何外部 API。Web 版用 WebGPU 加速或 WASM 降级，CLI 版支持 Node.js GPU 加速。

MCP
MCP 而非自定义插件协议

MCP 是 AI 工具领域的"USB 接口"。一套 MCP 服务器，Cursor、Claude Code、Codex、Windsurf 全部能用。不需要为每个编辑器写独立插件。

大白话

DeepWiki 帮你"理解"代码，生成文档描述。GitNexus 帮你"分析"代码——跟踪每一条依赖关系。

Sourcegraph 是云端文本搜索，基于关键词匹配。GitNexus 是本地图谱查询，基于语义关系。

CodeGraph 和 GitNexus 最接近，都是本地图谱引擎。GitNexus 多了浏览器端运行和预计算智能。

GitNexus 独有的杀手锏：在浏览器里就能跑完整的代码分析，不用装任何东西。

完整技术栈一览

CLI 和 Web 用同一套核心逻辑，不同的运行时适配：

🖥

CLI 运行时

Node.js 原生运行。Tree-sitter 原生绑定。LadybugDB 原生。Worker 线程池并行。本地文件系统持久化。

🌐

Web 运行时

浏览器 + WASM。Tree-sitter WASM 解析。LadybugDB WASM 存储。Web Workers + Comlink 并行。内存会话级存储。

🎨

Web 前端

React 18 + TypeScript + Vite 构建。Tailwind v4 样式。Sigma.js + Graphology (WebGL) 图可视化。i18n 多语言支持。

🔗

AI 接口

CLI 通过 MCP（stdio）暴露工具。Web 通过 LangChain ReAct Agent 提供对话。Bridge 模式让 Web 连接 CLI 后端获得完整能力。
