Zero-Server 架构：代码不出浏览器

为什么 GitNexus 能做到"代码永远不离开你的机器"？

两个包，一套逻辑

GitNexus 是一个 monorepo，包含两个核心包和共享类型库：

CLI
gitnexus/ — 核心引擎

CLI 命令行工具 + MCP 服务器 + 索引管道 + 图数据库。用 Tree-sitter 原生绑定解析代码，LadybugDB 原生模式存储图谱。Node.js 运行时。

WEB
gitnexus-web/ — 浏览器界面

React 18 + Vite + Tailwind v4 构建的可视化图谱浏览器。Tree-sitter WASM 解析代码，LadybugDB WASM 存储图谱，WASM 模式运行嵌入模型。一切在浏览器里完成。

SHARED
gitnexus-shared/ — 共享类型

TypeScript 类型和常量，CLI 和 Web 都用。支持 14 种语言的枚举、图谱 schema 定义、搜索接口等。

Zero-Server 理念的三个层次

🔒
隐私优先

CLI 模式全部本机运行，零网络请求。Web 模式全部浏览器内完成，API Key 只存在 localStorage。代码永远不离开你的机器。

🌐

解析层 — Tree-sitter WASM

Web 版把 Tree-sitter 编译成 WASM 在浏览器里运行。14 种语言的语法解析，全部在用户浏览器里完成，不调用任何外部 API。

🗃

存储层 — LadybugDB WASM

图数据库也编译成 WASM，在浏览器内存中运行。每次会话独立，关掉标签页数据自动清除。CLI 版则用原生 LadybugDB，持久化到 .gitnexus/ 目录。

🤖

嵌入层 — transformers.js

用 HuggingFace 的 transformers.js 在浏览器里生成文本嵌入向量（WebGPU 加速或 WASM 降级）。CLI 版用 Node.js 原生版本，支持 GPU 加速。

Bridge 模式：本地最强组合

CLI 和 Web 不是割裂的。通过 Bridge 模式，你可以两全其美：

Bridge 模式启动 

# 终端1：启动本地 HTTP 服务器
gitnexus serve

# 浏览器打开 gitnexus.vercel.app
# 页面自动检测本地服务器
# 所有 CLI 索引过的仓库直接可用
# 无需重新上传或重新索引
大白话

在终端启动 gitnexus serve，它会在 4747 端口开一个本地 HTTP 服务。

然后打开网页版 GitNexus，它会自动发现你本地的服务器。

所有你在 CLI 模式下索引过的仓库，在网页里直接浏览、搜索、问答。

代码不走外网——本地服务器和浏览器之间直接通信。
