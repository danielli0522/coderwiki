MCP Agent：工具调用引擎

MCP 是 AI 世界的"万能插座"——任何工具只要插上，AI 就能用。搜索引擎、文件操作、代码执行，统统变成 AI 的手和脚。

什么是 MCP？

MCP 让 AI 智能体能调用外部工具——搜索引擎、网页爬虫、文件操作等。Tarko 框架的 MCPAgent 专门处理这些工具调用。

MCPAgent 的代码实现

MCPAgent 继承自 Tarko 的基础 Agent 类，在初始化时连接所有 MCP Server，把它们的工具注册到自己身上。

CODE 

// multimodal/tarko/mcp-agent/src/mcp-agent.ts
export class MCPAgent<T>
  extends Agent<T> {
  private mcpClients:
    Map<string, IMCPClient>;

  async initialize() {
    for (const [name, config]
      of Object.entries(mcpConfig)) {
      const client =
        new MCPClientV2(name, config);
      await client.initialize();

      const tools = client.getTools();
      for (const tool of tools) {
        this.registerTool(
          new Tool({ id: tool.name,
            function: (args) =>
              client.callTool(tool.name, args)
          })
        );
      }
    }
  }
}
PLAIN ENGLISH

MCPAgent 继承自基础 Agent 类，专门处理 MCP 工具调用...

用一个 Map 管理所有 MCP 客户端连接...

初始化时遍历配置中的所有 MCP Server...

取出每个 Server 的名称和连接配置...

创建 MCP 客户端并建立连接...

获取这个 Server 提供的所有工具...

逐个注册为 Agent 可用的工具...

工具的执行函数就是调用 MCP 客户端的 callTool 方法...

传入工具名和参数，等待结果返回。

🔑
设计模式：依赖注入

MCPAgent 不知道也不关心它调用的工具是怎么实现的。它只通过 MCP 协议和工具通信。你可以换掉底层的搜索引擎、换掉爬虫实现——Agent 的代码一行都不用改。这就是 依赖注入 的威力。

Agent TARS：MCP + GUI 的融合

AgentTARS 是 MCP 能力的集大成者——它继承自 MCPAgent，同时集成了浏览器控制、文件系统操作和搜索能力。

CODE 

// multimodal/agent-tars/core/src/agent-tars.ts
export class AgentTARS<T>
  extends MCPAgent<T> {
  private readonly environment;

  constructor(options: T) {
    const env = options.aioSandbox
      ? new AIOEnvironment(options)
      : new LocalEnvironment(options);

    super({
      ...processedOptions,
      mcpServers:
        env.getMCPServerRegistry(),
    });
  }
}
PLAIN ENGLISH

AgentTARS 继承自 MCPAgent——它天生就有工具调用能力...

environment 负责管理运行环境（本地或沙箱）...

构造函数根据配置选择环境...

如果启用 AIO 沙箱，用隔离的云端环境执行...

否则用本地环境——浏览器、文件系统都在本机...

调用父类 MCPAgent 的构造函数...

关键：从 environment 获取 MCP Server 注册表...

这样 MCPAgent 就能连接所有环境提供的工具了。
