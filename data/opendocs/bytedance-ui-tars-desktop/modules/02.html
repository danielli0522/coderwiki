GUI Agent 循环：截图 → 思考 → 操作

想象一个棋手：看棋盘、思考下一步、落子——然后重复。GUI Agent 就是 AI 版的棋手，只不过棋盘是你的电脑屏幕。

核心循环：看 → 想 → 做

GUIAgent 类实现了 UI-TARS 最核心的循环。每一步都是：截取屏幕截图，交给视觉语言模型分析，模型返回"下一步该做什么"，然后执行操作。

1
截图（Screenshot）

用 Electron 的 desktopCapturer 捕获屏幕图像，转为 base64 编码

2
推理（Inference）

多模态 LLM 理解截图内容，输出 "Thought: ... Action: ..." 格式的指令

3
解析（Parse）

action-parser 把文字指令解析成结构化操作：点击坐标、输入文字、按键组合等

4
执行（Execute）

Operator（NutJS / 浏览器 / ADB）执行实际操作，然后回到第 1 步继续循环

GUIAgent 的循环代码

这段代码是 GUIAgent.run() 的核心——一个 while(true) 循环，不断截图、推理、执行，直到任务完成或达到最大循环次数。

CODE 

// packages/ui-tars/sdk/src/GUIAgent.ts
while (true) {
  if (loopCnt >= maxLoopCount) {
    data.status = StatusEnum.ERROR;
    break;
  }
  loopCnt += 1;

  // Step 1: 截图
  const snapshot =
    await operator.screenshot();

  // Step 2: 模型推理
  const { prediction,
    parsedPredictions } =
    await model.invoke(vlmParams);

  // Step 3: 执行每个操作
  for (const p of parsedPredictions) {
    if (p.action_type === 'finished')
      break;
    await operator.execute({
      parsedPrediction: p });
  }
}
PLAIN ENGLISH

进入无限循环——直到任务完成才会退出...

安全阀：如果循环次数超过 maxLoopCount（默认 25），强制停止...

状态设为 ERROR，跳出循环...

循环计数器加 1...

第一步：调用 operator 截取当前屏幕...

第二步：把截图和历史对话发给视觉语言模型...

模型返回预测结果和解析后的操作列表...

第三步：逐个执行模型建议的操作...

如果模型说 finished，说明任务完成了，跳出循环...

否则执行操作（点击、输入、滚动等），然后回到循环开头继续截图。

安全保护机制

循环不能无限运行——万一 AI 陷入死循环怎么办？UI-TARS 设计了多重保护。

GA
GUIAgent
循环控制
SC
截图 &
推理引擎
OP
操作执行
Operator
Click "Next Step" to begin
MAX_LOOP_COUNT = 25默认最大循环次数，防止 Agent 陷入死循环
MAX_SNAPSHOT_ERR_CNT = 10连续截图失败上限，超过则终止
AbortController用户随时可以点击停止，中断当前操作
