模型与 Prompt：AI 的眼睛和大脑

截图给谁看？AI 怎么理解屏幕上的元素？这背后是一套精心设计的 Prompt 模板和多版本适配系统。

模型适配层：UITarsModel

UI-TARS 支持多个视觉语言模型。UITarsModel 封装了 OpenAI SDK，统一了不同模型的调用方式。

CODE 

// packages/ui-tars/sdk/src/Model.ts
export class UITarsModel extends Model {
  async invoke(params: InvokeParams) {
    const { conversations, images,
      screenContext, uiTarsVersion } = params;

    // 压缩图片：不超过模型像素上限
    const compressed = await
      Promise.all(images.map(img =>
        preprocessResizeImage(
          img, maxPixels)));

    // 转换为 OpenAI 消息格式
    const messages =
      convertToOpenAIMessages(
        { conversations, images: compressed });

    // 调用模型并解析操作
    const { prediction } =
      await this.invokeModelProvider(...);
    const { parsed } = actionParser(
      { prediction, screenContext });
    return { prediction, parsedPredictions: parsed };
  }
}

Prompt 模板：教 AI 如何操作 GUI

不同版本的模型有不同的能力，所以 Prompt 也要适配。UI-TARS 为每个模型版本准备了专门的 System Prompt。

CODE 

// apps/ui-tars/src/main/agent/prompts.ts
export const getSystemPromptV1_5 = (
  language: 'zh' | 'en',
  useCase: 'normal' | 'poki'
) => `You are a GUI agent...
## Action Space
click(point='<point>x1 y1</point>')
drag(start_point=..., end_point=...)
type(content='xxx')
scroll(point=..., direction='down')
## Note
- Use ${language === 'zh' ? 'Chinese' : 'English'}
  in Thought part.`;

💡
模型版本适配

注意 V1.0 用 [x1, y1, x2, y2] 框选坐标，V1.5 用 <point>x y</point> 单点坐标，Doubao 20B 又回到了框选但增加了 press/release 组合键能力。每个模型的训练数据不同，Prompt 必须匹配模型的"语言"。

图片压缩：平衡精度与成本

一张 4K 截图可能有 800 万像素，但模型有像素上限。UI-TARS 会智能压缩图片，保留足够的细节让模型理解界面。

MAX_PIXELS_V1_0 = 1350 * 28 * 28V1.0 模型最大支持约 105 万像素
MAX_PIXELS_V1_5V1.5 模型支持更多像素，识别更精准
MAX_IMAGE_LENGTH = 5对话历史中最多保留 5 张图片，超出则滑动窗口裁剪
JPEG quality = 75截图用 75% 质量的 JPEG，在清晰度和 API 成本间取平衡
