A rehearsal studio for musicians — you get a fully equipped stage with every instrument (all AWS services) to practice on before the real concert (production AWS). No tickets needed, no security guards checking IDs.

Meet Floci

Floci is a local emulator that runs all of AWS on your machine. No AWS account needed, no credit card, no internet required.

The name comes from floccus — the cloud formation that looks like popcorn. Light, fluffy, everywhere you need it. That's exactly what Floci is: your own personal cloud, wherever you are.

1
Start the container

Run docker compose up in your terminal. One command, zero configuration.

2
Point your SDK to localhost:4566

That's the local port Floci listens on. Same API, same protocol, different address.

3
Develop freely

Create buckets, write items, invoke functions — everything works as if it were real AWS.

docker-compose.yml services: floci: image: floci/floci:latest ports: - "4566:4566" volumes: - ./data:/app/data
In plain English
Define a service called floci
Pull the latest Floci image from Docker Hub
Expose port 4566 so your code can reach Floci at localhost
Mount a local volume so your data survives restarts

Why Floci Exists

LocalStack Community sunset in March 2026 — it now requires auth tokens, and only receives frozen security updates.

Floci is the no-strings-attached alternative. Free, open-source, no account, no token, no limits.

47 Services, One Container

Floci supports 47 AWS services in a single Docker container. Here are the six you'll use most often:

📁
S3

File storage — upload and download files to buckets, like an infinite hard drive in the cloud.

🗃
DynamoDB

Database — store and query JSON-like data in tables, single-digit millisecond responses, no schema required.

⚡
Lambda

Functions — run code on demand without managing servers. Upload your function, trigger it, get results.

📩
SQS

Queues — send messages between parts of your app reliably. One service writes, another reads, no direct coupling.

🔒
IAM

Auth — control who can do what. Create users, roles, and policies to lock down access.

🔗
API Gateway

Routing — give your Lambda functions HTTP endpoints. REST APIs, WebSocket connections, all managed for you.

Plus 41 more: Step Functions, SES, SNS, Kinesis, EventBridge, CloudFormation, and many others — all running locally.
