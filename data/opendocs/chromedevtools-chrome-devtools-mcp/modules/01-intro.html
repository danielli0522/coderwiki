项目概览与 MCP 协议

Chrome DevTools MCP 是什么？它如何让 AI 编码助手「看见」并操控浏览器？

项目定位：AI 的浏览器遥控器

Chrome DevTools MCP（chrome-devtools-mcp）是一个由 Google 官方维护的 MCP 服务器。它让 Claude、Cursor、Copilot 等 AI 编码助手获得完整的 Chrome 浏览器操控能力 —— 包括点击、填表、截屏、网络分析、性能诊断、内存快照等。

项目拥有 42k+ GitHub Stars，是目前最流行的 MCP 服务器之一。核心价值：将 CDP 的复杂能力封装为标准化的 MCP 工具，让 LLM 只需发出 JSON 指令就能完成专业的浏览器调试。

项目文件结构

项目采用 TypeScript 编写，源码在 src/ 下按职责分层：

src/ 核心源码
index.ts MCP 服务器入口，注册工具与初始化
browser.ts 浏览器启动与连接管理
McpContext.ts 核心上下文：页面、网络、Console 收集器
McpPage.ts 单页状态封装：快照、对话、模拟设置
ToolHandler.ts 工具调度中间层：参数校验、互斥锁、遥测
tools/ 10 大类 45+ 工具定义
input.ts 点击、填表、拖拽、按键
pages.ts 导航、标签页管理
performance.ts 性能 Trace 录制与分析
network.ts 网络请求列表与详情
console.ts Console 消息与错误
memory.ts Heap Snapshot 分析
formatters/ 输出格式化（Console、网络、快照）
telemetry/ 使用统计与遥测

MCP 协议通信流程

整个系统基于 JSON-RPC 进行通信。AI 编码助手（MCP Client）发送工具调用请求，MCP Server 将其转发给 Chrome 浏览器执行，然后返回结果。

代码解读：服务器入口

index.ts 中的 createMcpServer 是一切的开始：

TypeScript export async function createMcpServer(serverArgs, options) { // 1. 初始化遥测日志 if (serverArgs.usageStatistics) { ClearcutLogger.initialize({persistence, logFile, appVersion}); } // 2. 创建 MCP Server 实例 const server = new McpServer({ name: 'chrome_devtools', title: 'Chrome DevTools MCP server', version: VERSION, }); // 3. 获取浏览器上下文（懒加载） async function getContext() { const browser = serverArgs.browserUrl ? await ensureBrowserConnected(...) : await ensureBrowserLaunched(...); if (context?.browser !== browser) { context = await McpContext.from(browser, logger, opts); } return context; } // 4. 注册所有工具 const tools = createTools(serverArgs); for (const tool of tools) { registerTool(tool); } return {server}; }
中文解读
创建 MCP 服务器的主函数，接收命令行参数和日志配置。
第一步：如果启用了使用统计，初始化 ClearcutLogger 遥测模块。
第二步：创建 McpServer 实例，声明名称和版本号。
第三步：定义 getContext 闭包，按需连接或启动浏览器。浏览器实例被缓存，避免重复启动。
第四步：调用 createTools 生成所有工具定义，然后逐个注册到 MCP 服务器。
