ECC 是什么 -- 一键安装的 AI 编程副驾驶

想象你有一个由 30 位专家组成的编程顾问团，ECC 就是让这些专家自动协作的操作系统。

从一行命令到 30 个专家同时待命

你正在用 Claude Code 写代码。你敲下一行命令 npx ecc install typescript，几秒钟后，你的项目里就多出了 30 个专业 Agent、135 个 Skill、60 个 Command。

ECC（Everything Claude Code）就是这样一个系统 -- 它把编程过程中需要的各种专业能力打包在一起，一键安装到你的开发环境里。

💡
核心理念

ECC 的哲学是 "Agent-First" -- 把工作尽早路由给最合适的专家。就像一个聪明的项目经理，不会自己写代码，而是找到最合适的工程师来干。

安装后的目录长什么样

安装完成后，你的项目里会出现这些文件：

.claude/ ECC 管理的所有配置
agents/ 30 个专业 Agent 定义文件
skills/ 135 个技能包 -- 工作流程和最佳实践
commands/ 60 个快捷命令 (/tdd, /plan 等)
hooks/ 自动化钩子 -- 在特定时机自动执行动作
rules/ 开发规范 -- AI 始终遵守的规则
📖
跨平台支持

ECC 不只支持 Claude Code。同样的配置可以安装到 Cursor、Codex、Gemini、OpenCode 等多种 AI 编程工具中。一套配置，多端复用。

核心配置长什么样

ECC 的身份定义在一个 YAML 文件中：

CODE 

spec_version: "0.1.0"
name: ecc
version: 2.0.0-rc.1
model:
  preferred: claude-opus-4-6
  fallback:
    - claude-sonnet-4-6
skills:
  - tdd-workflow
  - code-tour
  - security-review
中文解读

配置格式版本是 0.1.0 -- 未来可能会升级

这个系统的名字叫 ECC

当前版本 2.0.0 候选版 1 -- 接近正式发布

模型配置：优先用最强模型 Opus 处理复杂任务

如果 Opus 不可用，降级到 Sonnet 继续工作

这是降级策略，保证系统不会因为一个模型挂掉就停摆

启用的技能包列表 -- 测试驱动开发

代码导览 -- 帮新人理解代码库

安全审查 -- 自动检查代码中的安全隐患
