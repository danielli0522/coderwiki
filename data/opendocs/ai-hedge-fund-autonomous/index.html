自主交易的愿景

当 AI 接管华尔街的分析工作——这个项目究竟在做什么

想象一个 AI 投资团队

想象你雇了 19 位顶级投资分析师——沃伦·巴菲特、查理·芒格、迈克尔·伯里、凯西·伍德……每人有自己的投资哲学，分析同一支股票，然后一位投资组合经理综合所有意见做出最终交易决策。

ai-hedge-fund 就是这样的系统——但每个"分析师"都是一个 AI Agent，用代码模拟了传奇投资者的分析风格。

⚠️
重要免责声明

这个项目是教育和研究用途——不做真实交易，不是投资建议。它演示的是 AI Agent 协作的架构模式，不是炒股指南。

def run_hedge_fund(
    tickers: list[str],
    portfolio: dict,
    selected_analysts: list[str] = [],
    model_name: str = "gpt-4.1",
):
    workflow = create_workflow(selected_analysts)
    agent = workflow.compile()

        "messages": [HumanMessage(

        )]

    return final_state

等所有 Agent 跑完，返回最终状态（包含每个分析师的信号和最终决策）

从人类交易员到 AI 交易员

传统的量化交易系统用数学公式和统计模型做决策。AI 对冲基金则走了另一条路——让 AI 模拟人类分析师的思维过程。这两种方式有什么本质区别？

📊

传统量化交易

规则明确：如果 RSI < 30 且成交量 > 均值 2 倍，则买入。每条规则由人编写，模型只是执行器。优点是可解释，缺点是灵活性差——遇到新情况需要手动添加规则。

🤖

AI Agent 交易

意图驱动：「像巴菲特那样分析这只股票」。AI 理解投资哲学，综合财报、新闻、技术指标，输出有理由的判断。不写规则，而是给 AI 角色「人格」。

🔀

混合模式（本项目）

Agent 框架编排 + LLM 推理 + 结构化数据。技术指标用代码计算（精确），投资决策用 LLM 综合（灵活），风控用硬规则（安全）。

💡
为什么用 LangGraph 而不直接调 LLM？

如果你只是让一个 AI 分析股票，直接调 GPT-4 就行了。但当你需要 19 个 AI 同时工作、互相协作、按特定顺序执行、共享数据——就需要一个工作流引擎。LangGraph 就是这个引擎，它管理的是「谁来做什么、什么时候做、怎么传数据」。

代码翻译：Fundamentals Agent 的分析逻辑

以基本面分析师为例，看看一个分析 Agent 如何工作（来自 src/agents/fundamentals.py）：

CODE 

def fundamentals_analyst_agent(state, agent_id):
    financial_metrics = get_financial_metrics(
        ticker=ticker, end_date=end_date
    )
    return_on_equity = metrics.return_on_equity
    thresholds = [
        (return_on_equity, 0.15),
        (net_margin, 0.20),
    ]
    score = sum(m > t for m, t in thresholds)
    signal = "bullish" if score >= 2 else "bearish"

统计总分

得分 2 以上：看多（bullish）；否则：看空（bearish）

AgentState：Agent 之间的公告板

所有 Agent 通过 AgentState 传递信息。这个状态对象的结构展示了系统的数据模型：

AgentState 所有 Agent 共享的状态对象
messages[]LangChain 消息历史，让 LLM 知道对话上下文
data.tickers[]要分析的股票代码列表
data.portfolio当前持仓：{AAPL: 100股, cash: $50000}
data.analyst_signals所有分析师的信号：{buffett: {AAPL: bullish, confidence: 0.9}}
data.risk_analysis风险管理器的输出：{AAPL: {max_position: 50000, volatility: 0.18}}
metadata.show_reasoning是否打印每个 Agent 的推理过程（调试用）

信号冲突：当分析师们意见完全相反

真实的投资决策中，信号冲突是常态而非例外。当 Buffett 说强买、Burry 说强卖时，Portfolio Manager 不是简单地「投票计数」，而是进行加权推理：

CODE 

# Portfolio Manager 的决策逻辑（简化版）
def portfolio_decision(signals, risk_analysis):
    bull_count = sum(
        s.signal == "bullish"
        for s in signals
    )
    avg_confidence = mean(
        s.confidence for s in signals
        if s.signal == "bullish"
    )
    max_pos = risk_analysis.max_position

    if bull_count >= 0.6 * len(signals):
        qty = calc_quantity(
            max_pos, avg_confidence
        )
        return "BUY", qty
    elif bull_count <= 0.3 * len(signals):
        return "SELL", 0
    else:
        return "HOLD", 0
PLAIN ENGLISH

统计看多信号的数量

计算看多信号的平均置信度

（置信度越高，信号越可靠）

从风险管理器获取最大仓位限制

如果超过 60% 的分析师看多

根据置信度和仓位上限计算买入数量

决策：买入

如果不到 30% 看多

决策：卖出

介于 30%-60% 之间

决策：持有（观望）

⚠️
风险管理是硬约束，不是建议

注意代码里的 max_pos 变量——即使所有 19 个分析师都强烈看多，Portfolio Manager 也不能超过 Risk Manager 设定的仓位上限。这是系统的安全阀：理性分析很重要，但风险控制是不可逾越的红线。

在用真金白银之前，你需要验证 AI 对冲基金的决策是否真的有效。这就是回测的作用：

# 回测主循环（简化版）
def backtest(strategy, start, end):
    portfolio = {"cash": 100000}
    for date in date_range(start, end):
        signals = strategy.run(date)
        decisions = portfolio_mgr.decide(
            signals, portfolio
        )
        execute_trades(decisions)
        update_portfolio(portfolio)
    return calculate_metrics(portfolio)

📊
关键回测指标

年化收益率：一年的平均收益率，越高越好。最大回撤：从最高点到最低点的最大跌幅，越小越好。夏普比率：每承担一单位风险能获得多少收益，通常 >1 算不错，>2 算优秀。胜率：盈利交易占总交易的百分比。

系统扩展：如何添加新的 Agent

ai-hedge-fund 的架构让扩展变得简单。假设你想添加一个「宏观经济分析师」Agent，关注利率、通胀和 GDP 增长：

1
创建 Agent 文件

在 src/agents/ 下创建 macro_agent.py，定义分析逻辑和输出格式。输入是宏观数据，输出是 bullish/bearish/neutral 信号。

2
编写 System Prompt

在 src/prompts/ 下创建 macro_prompt.py，定义宏观分析师的投资框架——关注美联储利率、CPI、失业率等。

3
注册到工作流

在 create_workflow 函数里添加新节点，把它接入并行分析层。Portfolio Manager 会自动读取它的信号。

⚠️
扩展的黄金法则

新增 Agent 必须遵守「输入-输出契约」：从 AgentState 读取数据，写入标准格式的信号（ticker、signal、confidence、reasoning）。只要遵守这个契约，任何新 Agent 都能无缝接入系统——这就是共享状态架构的威力。
