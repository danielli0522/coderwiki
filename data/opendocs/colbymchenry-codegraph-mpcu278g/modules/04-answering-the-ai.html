The AI's Hotline: Model Context Protocol

When Claude Code, Cursor, or Codex wants to ask CodeGraph a question, it doesn't make an HTTP request to a web server. Instead, it communicates through MCP (Model Context Protocol).

Think of it as a private telephone line between the AI assistant and CodeGraph. When Claude sends a message, CodeGraph reads it from standard input, processes it, and writes the answer back to standard output — all in the same process, with zero network hops.

The Search Engine Inside: SQLite FTS5

The "instant search" you saw above is powered by FTS5, a full-text search extension built right into SQLite. CodeGraph creates a virtual FTS5 table over the nodes, and a trigger keeps it in sync automatically.

src/db/schema.sql (key parts) -- The main nodes table CREATE TABLE nodes ( id TEXT PRIMARY KEY, name TEXT, qualified_name TEXT, kind TEXT, file_path TEXT ); -- Virtual FTS5 search index CREATE VIRTUAL TABLE nodes_fts USING fts5(name, qualified_name, docstring, signature, content='nodes'); -- Auto-sync trigger on insert CREATE TRIGGER nodes_ai AFTER INSERT ON nodes BEGIN INSERT INTO nodes_fts ... END;
Plain English
The main table — one row per symbol
Each symbol has a unique ID
Its short name and its full dotted name
What kind of symbol (function/class/method) and where it lives
A "virtual" search table — like an index at the back of a book
Search across these four text fields simultaneously…
…with automatic ranking by relevance
Backed by the real nodes table (stays in sync)
Every time a node is inserted…
…the database automatically adds it to the search index too
No manual sync required — the database handles it
