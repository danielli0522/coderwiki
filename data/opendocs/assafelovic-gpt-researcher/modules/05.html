适合 vs 不适合

✅ 适合

  • 需要自动化市场调研的创业团队
  • 学术研究中的文献综述辅助
  • 竞品分析和行业趋势追踪
  • 需要定期生成研究报告的自动化流水线
  • 想学习 Agent 架构的 AI 工程师
  • 需要自建研究 Agent 而不想从零开始

❌ 不适合

  • 需要 100% 事实准确性的法律/医疗报告
  • 实时性要求极高的新闻快讯
  • 纯中文环境下搜索引擎支持有限
  • 不需要报告、只需要简短回答的场景
  • 对 LLM 成本极度敏感的应用

源码中的硬限制

以下是直接从源码中提取的关键常量和限制：

max_iterations = 3 默认最多搜索 3 轮。可在配置中修改，但更多轮次意味着更高的 LLM API 成本
max_subtopics = 5 子主题报告最多 5 个子主题。这是 GPTResearcher.__init__ 中的默认值
temperature = 0.15 Agent 选择时的温度参数。低温度确保选择结果稳定可复现
20+ 搜索引擎 retrievers/ 目录下有 20+ 搜索适配器，每个都需要对应的 API Key
Python 3.11+ 最低 Python 版本要求。因为使用了 match-case 语法（3.10+）和类型注解新特性
MIT License 开源许可证，可自由商用。但部分搜索引擎 API 可能需要付费

成本与结论

1
LLM API 成本

每次研究至少调用 LLM 5-8 次（Agent 选择、子问题生成、每个子问题的搜索、报告生成）。使用 GPT-4o 级别模型，单次研究约 $0.05-0.15。

2
搜索引擎 API 成本

Tavily 有免费额度（1000 次/月），Google Custom Search 每天 100 次免费。DuckDuckGo 免费但稳定性较差。

3
时间成本

单次完整研究约 30-90 秒，取决于并行搜索数量和网页爬取速度。深度研究模式需要 3-5 分钟。

4
自托管 vs 云服务

代码完全开源可自托管。也可用 Docker 一键部署。前端支持轻量 HTML 版本和生产级 Next.js 版本。

🎯
一句话结论

GPT Researcher 是目前最成熟的开源研究 Agent 框架。如果你的项目需要「自动搜索+自动写报告」的能力，它是最快上手的选择。但要注意：它的输出质量受 LLM 和搜索引擎双重限制，不适合对事实准确性要求极高的场景。
