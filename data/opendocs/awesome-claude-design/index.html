核心理念

DESIGN.md 是一个纯文本 Markdown 文件，描述品牌的视觉语言。把它交给 Claude Design，几分钟就能生成完整的设计系统：CSS 变量、字体方案、组件库、预览页面——全部搞定。

DESIGN.md 的典型结构预览

一个完整的 DESIGN.md 文件通常包含这些章节，每个章节都有明确的格式和内容要求：

视觉主题 用 2-3 个形容词描述整体感觉：如「精密、暗色、优雅」
配色方案 每个颜色的用途 + Hex 值 + 适用的状态场景
排版系统 字体家族 + 字号阶梯（xs/sm/base/lg/xl/2xl）+ 行高和字重
组件样式 按钮/输入框/卡片等组件的视觉规范和状态变化
布局规则 间距阶梯、网格系统、响应式断点
Do/Don't 明确的设计护栏——告诉 AI 什么是风格内的，什么不是
📝
为什么用 Markdown 而不是 JSON/YAML？

因为 Claude（和大多数 LLM）对 Markdown 的理解最自然。Markdown 既有结构（标题、列表、表格），又允许自然语言描述（设计理由、使用场景）。JSON/YAML 适合机器解析，但丢失了「为什么这样设计」的上下文。DESIGN.md 选择 Markdown 是为了最大化 AI 的理解深度，而非数据可解析性。

DESIGN.md 的 9 大结构

每个 DESIGN.md 都遵循这 9 个标准章节

从氛围到 Agent 提示词

1
视觉主题与氛围

定调：密集还是疏朗、深沉还是明亮、冷色还是暖色。这是整个系统的情绪基调。

2
配色方案与角色

定义每个颜色的用途——主色、背景、强调、成功/错误/警告。Claude 据此生成 CSS 变量。

3
排版规则

字体选择、字号阶梯、行高。Claude 会自动找 Google Fonts 替代方案（如果品牌字体是专有的）。

4
组件样式

按钮、输入框、卡片、导航——包含各种状态（默认、悬浮、激活、禁用）。

5
布局原则

间距阶梯、网格系统、留白节奏。决定了页面呼吸感。

还有 4 个关键章节

6
深度与层次

阴影 token 和表面层级。哪些元素浮在前面，哪些沉在后面。

7
应该做和不应该做

Claude 生成新页面时的护栏——"按钮不用渐变"、"标题不超过两行"之类的硬规则。

8
响应式行为

断点、触摸目标、折叠行为。确保在手机上也好用。

9
Agent 提示词指南

可复用的提示词模板，Claude 会嵌入到生成的 SKILL.md 中，方便未来项目复用。

DESIGN.md 的「三层信息密度」设计

每个章节都包含三个层次的信息：Token（具体值）如 --color-accent: #7B6DAA、Rule（规则）如「主按钮使用品牌色填充」、Rationale（原因）如「紫色传达创意与专业并重的品牌印象」。这种三层结构让 AI 既有精确指令（用哪个颜色），又有决策框架（什么时候用），还有上下文（为什么）——遇到 DESIGN.md 没覆盖的边缘情况，AI 也能根据 rationale 做出一致的选择。

Token

具体值层

精确的 CSS 变量、字号像素值、间距 token。AI 不需要猜测，直接使用。--spacing-sm: 8px、--font-heading: Bricolage Grotesque。

Rule

规则层

使用规则和约束条件。「标题只用 weight-700」「按钮圆角统一 8px」「卡片阴影不超过 2 层」。这些规则确保一致性。

Rationale

原因层

设计决策背后的思考。「紫色因为传达创意+专业」「大留白因为产品强调阅读体验」。这是 AI 能泛化的关键。

如果你想在未来项目中不用重新上传 DESIGN.md 就能复用相同的设计风格，应该依赖 DESIGN.md 的哪个章节？

输出文件详解：每个文件的作用

README.md 包含品牌上下文和语调指南，是整个系统的「为什么」文档。colors_and_type.css 包含所有 CSS 变量定义，可以直接在你的项目中 import 使用。preview/ 目录包含可视化的预览卡片，帮你快速确认颜色、字体、间距是否正确。index.html 是一个完整的营销页面 demo，展示了所有组件在真实布局中的效果。SKILL.md 是核心复用文件——包含精简的提示词模板，让未来项目一键召唤相同风格。

⚙️
DESIGN.md 的常见定制场景

下载的 DESIGN.md 不是最终版——你应该根据项目需求定制它。最常见的三种定制：(1) 替换品牌色——把 Stripe 的紫色换成你品牌的颜色，Claude 会自动推导出完整的色阶（浅色、深色、悬浮态等）；(2) 调整间距节奏——如果你的内容密度更高，把 spacing-base 从 16px 调到 12px，整个间距阶梯会重新计算；(3) 修改字体搭配——如果品牌字体不在 Google Fonts 上，Claude 会自动找最接近的替代方案。

定制示例：从 Stripe 到你的品牌 
# 原始（Stripe 风格）
Primary:   #635BFF  → 改为你的品牌色
Heading:   Bricolage Grotesque  → 改为你的字体
Spacing:   16px base  → 改为你的节奏

# Claude 自动推导的变化
--color-primary-hover   → 自动调深 10%
--color-primary-light   → 自动加白 90%
--spacing-sm/lg/xl      → 按 base 重新计算
定制要点

只需改 3 个核心值，Claude 推导整个系统

品牌色是影响最大的单一改动

字体替换会自动匹配 Google Fonts

间距改变会影响所有组件的节奏感

定制的 DESIGN.md 保存后可以反复复用

🎯
定制时的常见错误

不要只改颜色不改间距——颜色和间距是一体的。暗色主题通常需要更大的间距来保持呼吸感，而亮色主题可以更紧凑。如果你从 Stripe（亮色）切换到深夜模式，记得同步增大 spacing-base。

💡
验证定制效果的最佳方法

生成后不要只看 index.html demo——重点检查 preview/ 目录里的颜色卡片和组件卡片。这些独立的预览卡片能帮你快速发现配色不协调、间距不一致等问题，比看完整页面更容易定位问题。

进阶技巧：混合多种风格

你不是只能选一套 DESIGN.md——高级用法是把两三套风格的元素混合起来。例如：用 Linear 的间距和布局系统 + Spotify 的配色活力 + Stripe 的组件精致度。方法是在 DESIGN.md 的每个章节中选择性引用不同风格的规则。Claude 会自动处理冲突（当两个风格矛盾时，优先使用第一个引用的规则）。

🔧
调试不一致的输出

如果 Claude 生成的页面风格不一致，检查你的 DESIGN.md 是否有矛盾的规则。常见问题：第 2 章说「圆角 8px」但第 4 章说「按钮 sharp edges」——Claude 会随机选一个。解决方法：在「应该做/不应该做」章节中明确优先级。

📱
响应式断点的注意事项

DESIGN.md 第 8 章定义了响应式行为，但 Claude 生成移动端布局时可能过度简化。建议在 prompt 中明确说「保持桌面端的间距节奏，不要在移动端突然改变品牌感」。这样可以避免移动端看起来像另一个产品。

🔄
版本迭代策略

把 DESIGN.md 当代码管理——每次品牌升级都更新版本号，用 git 追踪变化。当你的设计语言进化时，只需要更新 DESIGN.md 然后让 Claude 重新生成，所有页面自动保持一致。这比手动改 50 个页面的 CSS 效率高几个数量级。
