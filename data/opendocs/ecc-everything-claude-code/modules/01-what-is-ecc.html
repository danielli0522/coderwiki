插件架构：一段配置看清全貌

ECC 的核心是一份 配置文件 plugin.json。它告诉 Claude Code："加载哪些 Hook、哪些 Agent、哪些 Command。"

plugin.json { "name": "everything-claude-code", "version": "2.0.0-rc.1", "description": "The ultimate Claude Code framework", "hooks": ["hooks/hooks.json"], "agents": ["agents/*.md"], "commands": ["commands/*.md"], "skills": ["skills/*.md"] }
中文解读
name — 插件的名字叫 "everything-claude-code"，即"万物 Claude Code"
version — 当前版本号 2.0.0-rc.1，rc 表示"发布候选版"
description — 一句话描述："终极 Claude Code 框架"
hooks — 指向 hooks.json，里面注册了各种自动化钩子
agents — 用通配符 *.md 加载 agents 目录下所有 60 个 Agent 定义
commands — 加载 commands 目录下所有 75 个命令定义
skills — 加载 skills 目录下所有 232 个技能定义

注意 *.md 这个通配符写法 — 它的意思是"加载这个目录下所有的 Markdown 文件"。每个 Agent、Command、Skill 都是一个独立的 .md 文件，这种设计让你可以灵活增减功能。

🔍
设计亮点：配置与代码分离

plugin.json 里的 hooks 字段指向的是 hooks/hooks.json（另一个配置文件），而不是直接写 Hook 代码。 这种"配置文件指向配置文件"的设计，让每个部分都能独立修改，互不影响。
