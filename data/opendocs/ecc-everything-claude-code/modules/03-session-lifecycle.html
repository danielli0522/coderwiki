03

会话生命周期

想象你入住一家酒店 —— 办理入住、享受住宿、退房结账。ECC 的会话管理也是这个流程，只不过"忠诚积分"让每次入住都比上一次更丝滑。

酒店入住 —— session-start 的五步仪式

你打开 Claude Code，ECC 的 session-start hook 就像前台接待员一样自动开始工作。它会帮你取出上次的"行李"，准备好所有服务。

整个过程不到一秒，但完成了五件关键的事：

1
Claude Code 启动，session-start.js 触发

Hook 机制监听到 session 启动事件，自动调用入口脚本。你什么都不用做。

2
加载上一次会话的摘要

从 持久化 存储中读取上次 context，让你从上次中断的地方继续。

3
恢复积累的"本能"（instincts）

加载你过去所有会话中提炼出的 instinct —— 你的编码偏好、常用模式、已知项目风格。

4
注册所有 hooks 和 agents

把 agent 和 checkpoint 等所有扩展能力挂载到当前会话的 lifecycle 中。

5
你可以开始编码了！

ECC 已经把"老客户的档案"全部准备就绪。你的 token 配置、历史偏好、安全策略都已生效。

session-start.js —— 读代码，看前台做了什么

下面是 scripts/hooks/session-start.js 的核心逻辑。左边是代码，右边是它每一行在"做什么事"：

CODE · session-start.js 
async function onSessionStart() {
  const context = await loadSessionContext();
  const instincts = await loadInstincts();
  const skills = await discoverSkills();

  if (context.lastSession) {
    console.log(`Resuming from: ${context.lastSession.id}`);
  }

  await registerHooks(hookConfigs);
  await loadAgents(agentPaths);

  return { context, instincts, skills };
}
中文翻译

定义异步函数 onSessionStart —— 会话启动时的总入口。

从磁盘加载上一次会话的 context（对话历史、项目信息）。

加载所有积累的 instinct（你的编码偏好和学到的模式）。

扫描 skillpack 目录，发现所有可用的 skill。

如果存在上一次会话记录的话...

打印日志：正在从某个会话 ID 恢复。

（条件结束）

把所有 hook 配置（安全、学习、检查点等）注册到生命周期中。

加载所有 agent 模块（安全 agent、学习 agent 等）。

返回打包好的三件套：context、instincts、skills，供后续使用。

📝
为什么要分三步加载？

context 是"你上次在做什么"，instincts 是"你一直喜欢怎么做"，skills 是"现在有什么工具可以用"。三者更新频率不同：context 每次会话变，instincts 偶尔积累更新，skills 只在你安装新技能包时才变。分开存储，关注点分离，读写更高效。

退房结账 —— session-end 的数据提炼

当你关闭 Claude Code，session-end hook 触发。它不只是关门，更像一位管家在整理你住过的房间 —— 提取有价值的模式，为下次入住做准备。

SM

Session Manager

总调度员。收集会话元数据（时长、token 用量、操作次数），决定哪些数据需要提炼、哪些直接归档。

PE

Pattern Extractor

模式挖掘机。用 AI 分析你的对话记录，提取可复用的行为模式，比如"遇到 React 报错时先检查 hooks 依赖数组"。

IDB

Instinct DB

长期记忆库。把新模式合并到你已有的 instinct 中，去除重复、保留最新。这是 ECC 越用越聪明的秘密。

SS

Summary Store

会话档案馆。保存完整的会话摘要，包括完成了什么任务、用了什么策略、遇到了什么问题。下次 session-start 会读取它。

💡
Aha! -- 你的经验从不浪费

你每次用 Claude Code 的经验都不会浪费。ECC 的会话系统就像一个从不忘记的助手，每次对话都让它变得更聪明。Session-start 恢复记忆，session-end 提炼经验，instinct DB 持续积累 —— 这个闭环让第 100 次对话比第 1 次精准得多。这就是 lifecycle 管理的真正价值。
