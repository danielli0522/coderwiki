持续学习引擎

想象你在健身房办了一张会员卡 —— 每次锻炼（会话）都在积累肌肉记忆（本能）。去的次数越多，力量越强。置信度分数就像你的个人记录 tracker，精确衡量每一次进步。

为什么持续学习很重要？

普通的 Claude Code 每次启动都是一张白纸 —— 没有任何记忆，不认识你，不了解你的项目。

ECC 的 instinct 系统让 Claude 从你的每一次交互中学习，越用越懂你。

🎨
代码风格偏好

学习你的格式化选择 —— 缩进用 tab 还是空格、分号加不加、单引号还是双引号。ECC 的 pattern recognition 让它自动适配你的风格。

🏗️
项目结构认知

记住你的架构决策 —— 组件放哪个目录、状态管理用什么方案、API 路由怎么组织。就像健身教练了解你的身体结构一样。

🔍
错误模式识别

更快发现重复出现的问题。如果你总是忘记处理某个 edge case，ECC 会主动提醒你。

🛠️
工具使用习惯

优化工具选择 —— 你更喜欢用 grep 还是 find？习惯先跑测试还是先写代码？ECC 会学着像你一样思考和行动。

💡
Aha! — 普通 AI 像金鱼

普通 AI 像金鱼，每次对话都是全新的。ECC 让 AI 像你的私人教练，记得你上次练了什么、哪些动作做得好、哪里还需要加强。

关键区别在于 feedback loop：每次你确认或纠正 ECC 的行为，它都在学习。这就是 machine learning 思想在开发工具中的落地。

Instinct 系统 —— 读代码，看"肌肉记忆"怎么存

每个 instinct 都是一个结构化对象。左边是代码，右边是它在"记录什么"：

javascript 
const instinct = {
  pattern: "prefer-typescript-strict-mode",
  confidence: 0.92,
  source: "observed-across-8-sessions",
  lastUpdated: "2024-01-15",
  appliesTo: ["new-files", "refactoring"]
};

if (instinct.confidence >= confidenceThreshold) {
  applyInstinct(instinct);
}
中文解读
pattern — 模式名称：这个用户偏好 TypeScript 严格模式
confidence — confidence score = 0.92，即 92% 的把握。经过 8 次会话观察得出。
source — 数据来源：在 8 个会话中反复观察到这个偏好
lastUpdated — 最后更新时间，确保使用最新数据
appliesTo — 适用范围：创建新文件和重构代码时才生效
 
如果置信度 >= threshold 阈值...
就自动应用这个 instinct —— 像肌肉记忆一样，不需要你再次确认。
🔍
为什么用 0.92 而不是 100%？

置信度永远不会到 1.0，因为人的偏好可能随时改变。0.92 代表"我非常确定，但留了 8% 的余地"。如果连续几次你开始用 JavaScript 而不是 TypeScript，置信度会自动下降。这就是 heuristics 的智慧 —— 不追求绝对正确，追求持续适应。
