五大角色

一部电影的幕后，由五种角色共同撑起。Claude Code 也是如此。

电影剧组的五种角色

ECC 的世界里，有五类「演员」在幕后协作。它们各有分工，共同确保每一次 prompt 都能得到精准响应。

D
Agent — 导演

做决策、分解任务、委派给其他角色。它是整个剧组的指挥中心。

S
Skill — 编剧

预写好的「剧本」，让 AI 面对常见场景时不用每次从零开始思考。

C
Command — 道具师

用户随时可以调用的斜杠命令，像道具一样即拿即用。

H
Hook — 舞台监督

在幕后默默运行，特定事件发生时自动介入。你不需要手动调用它。

I
Instinct — 演员记忆

从过往会话中学到的经验教训，帮助 AI 避免重复犯错、越用越聪明。

Aha!
为什么是五个而不是三个？

很多人以为 AI 只需要「理解 + 回答」。但 ECC 把能力拆成五个角色，就像电影不能只有导演和演员——编剧写剧本、道具师准备工具、舞台监督管流程、演员积累经验。每个角色专注一件事，组合起来才能应对复杂场景。这就是配置即能力的精髓。

深入 Agent：导演的剧本

一个 Agent 的配置文件 长什么样？下面是一段真实的 agents/planner.md 代码。

markdown --- name: planner description: > Strategic planning agent that breaks down complex tasks into actionable steps. Coordinates with other agents and delegates subtasks. directives: - Always think before acting - Break complex problems into steps - Delegate to specialized agents ---
中文解读
name: planner — 这个 Agent 叫「规划师」
description — 它是战略规划型代理，负责把复杂任务拆解为可执行的步骤，并与其他 Agent 协调
directives — 行为准则（导演的执导原则）：
  1. 行动前先思考
  2. 把复杂问题分解成步骤
  3. 把子任务委派给专业 Agent

Markdown 文件开头的 --- 区域叫 frontmatter， 用 YAML 语法定义了 Agent 的「身份」和「行为准则」。

Aha!
directives 就是导演的「执导原则」

注意那三条 directives——它们不是代码逻辑，而是用自然语言写的「行为约束」。AI 会把这些原则内化为决策依据。你写什么，它就倾向什么。这就是 directive 的力量。

五个角色的群聊

想象一下：用户执行了一个操作，五个角色在后台如何协作？点击下面的按钮，看一场实时「群聊」。

H
Hook（舞台监督）

用户刚执行了一个工具调用，我需要检查一下安全性...

A
Agent（导演）

好的，我来分析这个操作的风险等级。根据 confidence score，这属于中等风险。

I
Instinct（演员记忆）

根据上次类似操作的经验，这个操作通常是安全的。上次会话里我们处理过类似场景。

S
Skill（编剧）

如果需要，我可以启动安全扫描流程。相关的模板已经准备好了。

C
Command（道具师）

用户随时可以输入 /scan 来手动触发检查。我就站在旁边，随叫随到。

H
0 / 5 messages
Aha!
Hook 永远是第一个发言的

注意到没有？群聊中 Hook（舞台监督）总是第一个检测到事件并开口说话。这就是 Hook 的本质——它是事件驱动的，不需要你手动调用。事件发生，它就行动。

角色匹配挑战

根据场景描述，选择最合适的角色。

1. 你想让 AI 自动在每次保存文件前检查代码风格。你应该用哪个角色？

2. 你有一个复杂的多步骤任务，需要分解和委派。你该调用什么？

3. 你发现 AI 总是犯同一个错误。哪个角色能帮它记住教训？
