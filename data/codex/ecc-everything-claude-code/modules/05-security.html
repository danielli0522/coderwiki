安全防线

机场安检的 X 光机扫描行李，AgentShield 扫描代码。每一次 AI 操作都要过安检，才能抵达目的地。

AI 编程的安全威胁 —— 为什么机场安检不是摆设？

AI 编程工具能执行代码、修改文件、访问 API — 能力越大，风险越大。

最大的威胁叫 Prompt Injection ：恶意指令藏在代码注释、文件名或数据里，AI 一不小心就会执行。

ECC 的 AgentShield 就像机场安检系统 — X 光扫描仪（扫描威胁）、护照检查（验证身份）、禁飞名单（拦截已知危险模式）、安全须知（提升用户意识）。

1
恶意代码注入

攻击者在 prompt 中藏入 eval()、exec() 等指令，诱导 AI 执行危险操作。就像行李箱里藏了违禁品。

2
文件系统越权

AI 读写了不该碰的文件 — .env 密钥、系统配置、用户隐私数据。就像有人误入了机场的禁区。

3
数据泄露

AI 把 DATABASE_URL、API 密钥等敏感信息输出到日志或代码中。就像安检时你的护照信息被广播了出来。

4
供应链攻击

让 AI 安装了一个被篡改的 依赖包 ，攻击者通过它渗透你的项目。就像通过了安检的包裹里其实藏着违禁品。

AgentShield 配置 —— 读代码，看安检机器怎么运转

AgentShield 的核心是 PreToolUse Hook — 它在 AI 每次使用工具前拦截，就像 X 光机在行李上飞机前扫描。

hooks/hooks.json { "PreToolUse": [{ "matcher": "Write|Edit", "hooks": ["scripts/hooks/security-scan.js"], "options": { "scanPatterns": [ "eval(", "exec(", "child_process", "fs.unlink", "rm -rf" ], "blockOnThreat": true, "logLevel": "warn" } }] }
中文解读
PreToolUse — 在 AI 使用工具"之前"触发，这是安检的第一道闸门
matcher: "Write|Edit" — 只监听"写文件"和"编辑文件"这两种操作，因为它们最容易注入恶意代码
hooks — 指向安全扫描脚本，它就是那台 X 光机
scanPatterns — 禁飞名单！列出所有危险模式：eval()、exec()、child_process
blockOnThreat: true — 发现威胁立刻拦截，不让 AI 执行这个操作（相当于"禁止登机"）
logLevel: "warn" — 把安全事件记录到日志中，方便事后审查（相当于安检的监控录像）
🔍
设计亮点：关注点分离

配置文件（hooks.json）只定义"什么时候触发"和"用什么规则"，具体扫描逻辑在 security-scan.js 中。 这就像安检规定（"哪些东西违禁"）和 X 光机器（"怎么检测"）是分开的一样 — 规则可以随时更新，不用换机器。

关键洞察：安全扫描发生在 AI "行动之前"（Pre），而不是"行动之后"（Post）。这就是 纵深防御 的核心理念 — 在威胁到达目标之前就把它拦住。

💡
Aha! — AI 时代最大的安全威胁

在 AI 编程时代，最大的安全威胁不是黑客，而是你让 AI 处理的代码里藏着的恶意指令。 AgentShield 就像代码世界的杀毒软件，但它扫描的不是病毒，而是 意图。

记住：代码本身没有善恶，但意图有。AI 能执行代码，却无法判断意图 — 这就是为什么需要 AgentShield 在中间做"安检"。
