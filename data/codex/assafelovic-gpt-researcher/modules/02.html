认识核心角色：七大组件各司其职

就像一部电影有不同的演员，GPT Researcher 也有不同的组件扮演不同角色。

项目目录结构

先看看代码是怎么组织的。就像了解一个公司的部门设置一样，目录结构能帮你快速理解系统分工：

gpt_researcher/ 核心研究引擎
agent.py 主控类 GPTResearcher，系统的 CEO
actions/ 具体动作：搜索、爬虫、生成查询
skills/ 高级技能：研究执行、报告写作、深度研究
retrievers/ 20+ 搜索引擎适配器
scraper/ 网页爬虫工具箱
memory/ 向量存储和记忆管理
config/ 配置管理
multi_agents/ 多 Agent 协作模式（高级）

七个核心角色

GPT Researcher 内部有七个关键组件。把每个组件想象成一个团队成员：

🎯
GPTResearcher（主控）

团队 CEO。创建并协调所有其他组件。你只跟它打交道。

🔍
ResearchConductor（搜索指挥）

研究经理。规划搜索策略、生成子问题、协调搜索引擎并行工作。

📝
ReportGenerator（报告写手）

资深撰稿人。把收集到的信息整理成结构化的研究报告。

🧠
ContextManager（上下文管家）

资料管理员。把爬取到的内容做 向量嵌入，找到跟研究问题最相关的内容。

🕷️
BrowserManager（浏览器管家）

浏览器操作员。负责打开网页、提取正文内容、过滤广告和无关信息。

✂️
SourceCurator（素材筛选）

质量审核员。对搜索结果做排序和筛选，去掉重复和低质量来源。

🔬
DeepResearchSkill（深度研究）

进阶研究员。可选组件，用于更深入的多轮迭代研究。

组件是怎么创建出来的？

看看 agent.py 的 __init__ 方法——GPTResearcher 在初始化时就创建了所有团队成员：

CODE 
# Initialize components
self.research_conductor = ResearchConductor(self)
self.report_generator = ReportGenerator(self)
self.context_manager = ContextManager(self)
self.scraper_manager = BrowserManager(self)
self.source_curator = SourceCurator(self)
self.deep_researcher = DeepResearchSkill(self)
白话翻译

注释：初始化各个组件

雇佣搜索经理，把自己（self）传给它，让它能调用其他资源

雇佣报告写手，同样传入自己的引用

雇佣上下文管家

雇佣浏览器操作员

雇佣素材筛选员

雇佣深度研究员（仅在深度研究模式下）

🧩
设计模式：组合优于继承

GPT Researcher 没有把所有功能写在一个巨大的类里，而是把不同职责拆成独立的 Skill 类，每个都接收 self（主控的引用）。这叫「组合模式」——像搭积木一样组合功能。如果你想加新功能，只需要创建新的 Skill 类，不用改动主控代码。

进阶：多 Agent 协作模式

除了单 Agent 模式，GPT Researcher 还有一个更强大的 多 Agent 模式。在 multi_agents/ 目录下，有一个用 LangGraph 编排的完整研究团队：

🔍

ResearchAgent

负责执行初始研究，收集原始信息。是团队的「一线调查员」。

📋

EditorAgent

负责规划研究和分配任务。是团队的「主编」，决定研究哪些方向。

✍️

WriterAgent

根据研究结果撰写报告。是团队的「专栏作家」。

👥

HumanAgent

人类审核环节。可以批准或修改研究计划，实现人机协作。

🚀

PublisherAgent

最终发布报告。负责格式化输出和保存文件。

# multi_agents/agents/orchestrator.py
workflow = StateGraph(ResearchState)
workflow.add_node("browser", agents["research"].run_initial_research)

白话翻译

创建一个状态图（工作流）来编排 Agent

第 1 步：ResearchAgent 做初始调研

第 2 步：EditorAgent 规划研究计划

第 3 步：EditorAgent 并行执行研究任务

第 4 步：WriterAgent 撰写报告

第 5 步：PublisherAgent 发布报告
