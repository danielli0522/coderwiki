主要角色：goose 里谁负责什么

把系统想成一个协作团队，避免让 AI 把逻辑塞错位置。

四类角色

入口

CLI、Web UI、API 或 SDK，负责把人的意图变成结构化请求。

编排

把任务拆开，决定调用模型、工具、索引还是插件。

能力

模型推理、网页抓取、向量搜索、文件读写、浏览器调试等实际能力。

状态

配置、缓存、会话、向量库、任务历史，决定系统能不能稳定复盘。

CODE
class gooseService:
    def plan(self, request):
        return self.router.choose_next_step(request)

    def execute(self, step):
        return self.tools.run(step)
中文解释

Service 不直接等于模型，它通常承担业务协调职责。

plan 决定下一步是什么，这就是编排。

execute 把步骤交给工具层，避免一个文件什么都干。

让 AI 给 goose 加一个新能力时，最应该说清楚什么？
