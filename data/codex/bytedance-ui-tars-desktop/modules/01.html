UI-TARS 是什么：让 AI 看懂屏幕

想象你对着电脑说"帮我在携程订一张北京到上海的机票"——AI 看着屏幕，找到按钮、填写表单、点击提交。这不是科幻，这就是 UI-TARS。

这个项目解决什么问题？

传统自动化需要写死每个操作步骤——"点击坐标(320, 480)"、"在 id 为 username 的框里输入文字"。一旦界面改版，脚本全废。UI-TARS 换了个思路：让 AI 看懂屏幕，像人一样理解和操作。

💡
核心洞察

UI-TARS 的哲学是：与其为每个应用写专门的 API 集成，不如让 AI 直接"看"屏幕，理解界面元素，像人一样操作。这样任何应用都能用，不需要逐个适配。

🖥️

Agent TARS

多模态 AI Agent 框架，支持 CLI 和 Web UI，能通过视觉能力操作电脑和浏览器

🎮

UI-TARS Desktop

Electron 桌面应用，提供本地和远程的电脑/浏览器 GUI 操作能力

🧩

Tarko 框架

底层引擎，提供 Agent 循环、事件流、MCP 工具调用、LLM 通信等核心能力

项目结构一览

UI-TARS 是一个 monorepo，包含两大产品线和一个核心引擎层。

UI-TARS-desktop/ 整个项目根目录
apps/ui-tars/ Electron 桌面应用（主进程 + 渲染进程）
multimodal/agent-tars/ Agent TARS 核心（CLI + Web UI + Core）
multimodal/tarko/ 底层引擎：Agent 循环、MCP、LLM、事件流
multimodal/gui-agent/ GUI 操作符（NutJS、ADB、浏览器、AIO）
packages/ui-tars/sdk/ UI-TARS SDK：截图→分析→操作循环

"你好，世界"：GUIAgent 的入口

所有复杂的 GUI Agent 操作，最终都归结为一个循环：截图→模型推理→执行操作。这段代码是 UI-TARS Desktop 中创建智能体的入口。

CODE 

// apps/ui-tars/src/main/ipcRoutes/agent.ts
export class GUIAgentManager {
  private static instance:
    GUIAgentManager;
  private currentAgent:
    GUIAgent<Operator> | null;

  public setAgent(
    agent: GUIAgent<Operator>
  ) {
    this.currentAgent = agent;
  }
}
PLAIN ENGLISH

GUIAgentManager 是一个单例——全局只有一个实例...

它持有当前正在运行的 GUIAgent...

GUIAgent 使用泛型 Operator——可以是 NutJS、浏览器等不同的操作方式...

setAgent 方法把创建好的智能体存起来...

后续可以 pause/resume/stop 控制 Agent 的执行。

这个设计让桌面应用能统一管理 Agent 生命周期。
