架构选型：UI-TARS 技术决策

技术负责人视角——评估 UI-TARS 是否适合你的项目，以及它与竞品的核心差异。

核心定位

Multimodal AI Agent视觉+语言的多模态智能体框架——AI 能"看懂"屏幕并操作
GUI + MCP 双引擎GUI Agent 处理界面操作，MCP Agent 处理工具调用，按需组合
Tarko Agent Framework底层事件驱动的 Agent 循环框架，支持流式、重放、快照
Desktop + CLI + Web + Server四种部署形态：桌面应用、命令行、Web UI、HTTP Server
Apache-2.0 License开源可商用，无限制

适合 vs 不适合

✓ 适合

  • ✓ 自动化操作没有 API 的桌面应用或网站
  • ✓ 界面经常变化、难以维护固定选择器的自动化
  • ✓ 需要跨平台（桌面+浏览器+手机）混合操作
  • ✓ AI 驱动的 RPA 场景——客服、数据录入、流程自动化
  • ✓ 快速原型验证——用自然语言描述任务即可执行
  • ✓ 可接受 LLM API 调用成本的生产环境

✗ 不适合

  • ✗ 需要毫秒级精确操作的高频交易等场景
  • ✗ 大规模批量数据处理（效率不如专用脚本）
  • ✗ 预算有限无法承担 LLM API 费用
  • ✗ 需要 100% 确定性的回归测试（AI 有随机性）
  • ✗ 完全离线/无网络的环境
  • ✗ 涉及高度敏感操作（金融交易）无人监督的自动化
⚠️
成本估算

GUI Agent 每次循环（截图+模型推理）是一次多模态 API 调用。复杂任务需要 10-50 次循环。以 Claude 3.5 Sonnet 为例，单次截图约 1000 tokens 输入 + 200 tokens 输出，一个 30 步任务大约消耗 $0.1-0.5。生产使用前务必评估成本。

源码中的硬限制

MAX_LOOP_COUNT = 25单次任务最大循环次数，防止 Agent 无限运行（来源：GUIAgent.ts）
MAX_SNAPSHOT_ERR_CNT = 10连续截图失败上限（来源：constants.ts）
MAX_IMAGE_LENGTH = 5对话历史最大保留图片数，超出的老图片会被裁剪（来源：utils.ts）
maxIterations = 1000Tarko Agent 框架默认最大迭代次数（来源：agent.ts）
maxTokens: V1.0=1000, V1.5=65535不同模型版本的最大输出 token 数（来源：Model.ts）
Node.js >= 22Agent TARS CLI 要求 Node.js 22 或更高版本（来源：README.md）
