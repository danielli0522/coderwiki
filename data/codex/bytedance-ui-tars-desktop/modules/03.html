Operator：从 AI 指令到鼠标点击

AI 说"点击(335, 360)"，但电脑怎么知道你要点哪里？Operator 就是 AI 世界和真实电脑之间的翻译官。

Operator 的角色

模型输出的是文字指令——"click(start_box='[0.131, 0.25, 0.131, 0.25]')"。但电脑需要真实的鼠标移动和键盘输入。Operator 就是这个翻译层。

🖱️

NutJS Operator

用 NutJS 库控制真实鼠标键盘，适用于本地桌面操作

🌐

Browser Operator

控制浏览器内的 DOM 操作，支持 Playwright 和 CDP

📱

ADB Operator

通过 Android Debug Bridge 控制安卓手机界面

Electron 桌面截图的实现

在 Electron 桌面应用中，截图不是用浏览器 API，而是用 Electron 原生的 desktopCapturer。这段代码展示了如何获取高质量屏幕截图。

CODE 

// apps/ui-tars/src/main/agent/operator.ts
export class NutJSElectronOperator
  extends NutJSOperator {

  public async screenshot() {
    const { physicalSize, logicalSize,
      scaleFactor } = getScreenSize();

    const sources = await
      desktopCapturer.getSources({
        types: ['screen'],
        thumbnailSize: {
          width: logicalSize.width,
          height: logicalSize.height }
      });

    const screenshot =
      sources[0].thumbnail;
    return {
      base64: screenshot.toJPEG(75)
        .toString('base64'),
      scaleFactor
    };
  }
}
PLAIN ENGLISH

继承 NutJSOperator，扩展 Electron 特有的截图能力...

获取屏幕的物理尺寸、逻辑尺寸和缩放比例...

调用 Electron 的 desktopCapturer API 获取屏幕截图...

只截取屏幕（不是窗口）...

按逻辑尺寸获取缩略图（考虑 Retina 缩放）...

取第一个屏幕源（主显示器）...

转为 JPEG 格式（质量 75%），再转 base64 文本...

返回 base64 图片和缩放因子——供模型使用。

Windows 输入的特殊处理

在 Windows 上直接用 keyboard.type() 输入中文会出问题。UI-TARS 用了一个聪明的替代方案：先复制到剪贴板，再 Ctrl+V 粘贴。

AI
AI 模型

我需要在这个搜索框里输入"北京到上海的机票"——执行 type(content="北京到上海的机票")

PA
Action Parser

解析完成：action_type = type, content = "北京到上海的机票"

OP
NutJS Operator

检测到是 Windows 系统，keyboard.type() 不支持中文。启用剪贴板方案...

OS
操作系统

clipboard.writeText("北京到上海的机票") → Ctrl+V → 输入完成！原剪贴板内容已恢复。

Action Space：AI 可以做哪些操作？

模型的输出格式被限定在一组预定义的"动作空间"里。就像给棋手规定只能下在棋盘交叉点上一样。

CODE 

// 定义 AI 可用的操作列表
static MANUAL = {
  ACTION_SPACES: [
    "click(start_box='[x1,y1,x2,y2]')",
    "drag(start_box=..., end_box=...)",
    "type(content='xxx')",
    "scroll(start_box=..., direction=...)",
    "hotkey(key='ctrl c')",
    "finished()",
    "call_user()",
  ]
};
PLAIN ENGLISH

定义了 AI 能执行的所有操作类型...

click：点击某个区域（用坐标框表示）...

drag：从一个区域拖拽到另一个区域...

type：在当前位置输入文字内容...

scroll：在某个区域向指定方向滚动...

hotkey：模拟快捷键组合（如 Ctrl+C）...

finished：任务完成，停止循环...

call_user：遇到困难，请求用户帮助。
