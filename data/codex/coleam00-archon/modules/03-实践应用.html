状态（State）：信息的载体

在 Archon 中，所有角色共享一个「状态」对象。你可以把它想象成一个游戏存档——每个角色都可以查看和修改它，下一个角色看到的是上一个人更新后的版本。

CODE 

class AgentState(TypedDict):
    user_request: str
    plan: Optional[str]
    generated_code: Optional[str]
    review_feedback: Optional[str]
    iterations: int
PLAIN ENGLISH

定义一个名为 AgentState 的数据结构。

user_request：用户的需求描述（文本）。

plan：推理者制定的计划（可选，一开始为空）。

generated_code：编码者生成的代码（可选）。

review_feedback：审查者的反馈意见（可选）。

iterations：迭代次数，记录优化了多少轮。

💡
关键洞察：为什么用共享状态？

共享状态就像一个共享文档（比如 Google Docs）。所有角色都能看到最新内容，也能添加自己的贡献。这样每个角色都不需要直接和其他角色对话——它们只需要读写状态就行了。这大大简化了系统的设计。

工作流图：连接各个节点

LangGraph 的工作流图不仅定义了有哪些角色（节点），还定义了它们之间的连接方式（边）。就像地铁线路图不仅标注了站点，还画出了线路走向。

CODE 

workflow.add_edge(START, "reasoner")
workflow.add_edge("reasoner", "coder")
workflow.add_conditional_edges(
    "reviewer",
    should_continue,
    {"continue": "coder",
     "finish": END}
)
PLAIN ENGLISH

从起点出发，第一个经过推理者。

推理者完成后，流向编码者。

审查者之后是条件分支——

根据审查结果决定下一步。

如果需要继续优化，返回编码者。

如果质量达标，结束工作流。
