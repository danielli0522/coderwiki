认识核心角色：Archon 的演员表

每个系统都有自己的「演员」，Archon 的演员们各司其职

Archon 里的角色们

想象一部电影的拍摄现场。有导演、编剧、演员、剪辑师——每个人负责不同的事情，但都为了同一部电影服务。Archon 也一样，它有几个核心「角色」，每个角色各司其职。

🧠
Reasoner（推理者）

像导演一样统筹全局——分析用户需求，制定行动计划，决定下一步做什么

💻
Coder（编码者）

像编剧一样写出具体的「剧本」——根据推理者的计划生成代码

🔍
Reviewer（审查者）

像剪辑师一样把关质量——检查生成的代码有没有问题，提出修改建议

📚
Knowledge Base（知识库）

像道具仓库一样提供背景资料——存储最佳实践和参考代码供其他角色使用

角色之间的对话

当你说「帮我创建一个能管理待办事项的 Agent」时，Archon 的角色们会这样交流：

👤
用户

帮我创建一个能管理待办事项的 Agent，它要能添加、删除、查询任务。

🧠
推理者

收到需求。我需要分析一下：这个 Agent 需要哪些工具？我判断需要 CRUD 操作工具和一个状态管理系统。

🧠
推理者

编码者，请你根据这个设计生成代码。需要用到 LangGraph 的 StateGraph。

💻
编码者

好的，我来写代码。我会创建状态模型、定义工具函数、搭建工作流图。

💻
编码者

代码生成完毕！审查者，请帮我检查一下质量。

🔍
审查者

检查完毕。第 45 行的错误处理不够完善，建议增加重试机制。其他部分通过！

深入推理者（Reasoner）

推理者是整个系统的「大脑」。它的职责是理解用户需求，然后拆解成具体的执行步骤。

CODE 

def reasoner(state: AgentState):
    """分析需求，制定计划"""
    user_request = state["user_request"]
    plan = llm.invoke(
        f"为以下需求制定执行计划:
        {user_request}"
    )
    state["plan"] = plan
    return state
PLAIN ENGLISH

定义推理者函数，接收当前的 Agent 状态。

注释说明功能：分析需求并制定计划。

从状态中取出用户的需求描述。

调用大语言模型（LLM），让它分析需求。

告诉 LLM：「为这个需求制定计划」。

把 LLM 的回应传入。

把制定好的计划保存回状态中。

返回更新后的状态给下一步使用。

项目的文件结构

Archon 的代码组织得非常清晰。下面是它的主要目录结构——每个文件夹都有自己的职责。

archon/ 项目根目录
graph/ LangGraph 工作流定义——Agent 的「大脑」在这里
services/ 与外部服务交互的代码（如 OpenAI API）
models/ 数据模型定义——描述信息的格式和结构
utils/ 工具函数——可复用的辅助代码
archon.py 主入口文件，启动整个系统
