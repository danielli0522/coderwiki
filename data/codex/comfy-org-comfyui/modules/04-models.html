模型管家：显存、缓存与智能调度

GPU 显存是稀缺资源，ComfyUI 如何聪明地管理它？

显存：AI 绘图最稀缺的资源

生成一张 AI 图片需要加载好几个模型——一个主模型（几 GB）、一个文本编码器（几百 MB）、一个 VAE 解码器——全都得塞进 GPU 显存 里。

如果你的显卡只有 8GB 显存，而模型需要 10GB，就会报 OOM（Out of Memory） 错误。

ComfyUI 的 model_management 模块就像一个精明的仓库管理员，决定哪些模型该上架、哪些该先收起来。

显存管理的关键代码

ComfyUI 在执行完每个节点后，会自动做显存清理：

CODE 

if comfy.model_management.is_oom(ex):
    tips = "This error means you ran out of memory on your GPU."
    comfy.model_management.unload_all_models()

# 正常执行完后的清理
if need_gc:
    gc.collect()
    comfy.model_management.soft_empty_cache()

模型文件怎么找到的？

folder_paths.py 是 ComfyUI 的文件导航系统。它把各种模型类型映射到对应的文件夹：

CODE 

folder_names_and_paths["checkpoints"] = (
    [os.path.join(models_dir, "checkpoints")],
    supported_pt_extensions
)
folder_names_and_paths["loras"] = (
    [os.path.join(models_dir, "loras")],
    supported_pt_extensions
)
folder_names_and_paths["custom_nodes"] = (
    [os.path.join(base_path, "custom_nodes")],
    set()
)

实用技巧

你可以通过 extra_model_paths.yaml 文件添加额外的模型目录。这样你的模型可以放在外部硬盘或其他位置，而不用全部塞进 ComfyUI 的 models 文件夹。ComfyUI 启动时会自动读取这个配置文件。
