节点是怎么注册进系统的？

每个节点都是一个 Python class。ComfyUI 用一个全局字典 NODE_CLASS_MAPPINGS 来管理它们：

CODE 

class CLIPTextEncode(ComfyNodeABC):
    @classmethod
    def INPUT_TYPES(s):
        return {"required": {
            "text": (IO.STRING, {"multiline": True}),
            "clip": (IO.CLIP, {})
        }}
    RETURN_TYPES = (IO.CONDITIONING,)
    FUNCTION = "encode"
    CATEGORY = "conditioning"

    def encode(self, clip, text):
        tokens = clip.tokenize(text)
        return (clip.encode_from_tokens_scheduled(tokens),)

通用模式

这个 INPUT_TYPES + RETURN_TYPES + FUNCTION 的三件套是 ComfyUI 节点的标准模板。几乎所有节点（包括你写的自定义节点）都遵循这个约定。学会这个模式，你就能读懂任何一个节点。

自定义节点：社区的力量

ComfyUI 最强大的地方之一就是自定义节点系统。任何人都可以写一个 Python 模块 放到 custom_nodes/ 目录，ComfyUI 会自动发现并加载它：

CODE 

node_paths = folder_paths.get_folder_paths("custom_nodes")
for custom_node_path in node_paths:
    possible_modules = os.listdir(custom_node_path)
    for possible_module in possible_modules:
        # 跳过禁用的和缓存文件
        if module_path.endswith(".disabled"):

概念速查

NODE_CLASS_MAPPINGS 全局字典，把节点类名映射到实际的 Python class 对象——是节点的'花名册'
INPUT_TYPES 类方法，声明节点需要什么输入（类型、默认值、约束）
RETURN_TYPES 元组，声明节点输出什么类型的数据——决定了连线时的类型匹配
FUNCTION 字符串，指定真正执行逻辑的方法名——ComfyUI 会调用这个名字的方法
CATEGORY 节点在菜单中的分类路径，如 "conditioning" 或 "image/upscaling"
