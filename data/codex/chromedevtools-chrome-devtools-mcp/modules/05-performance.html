CrUX 真实用户数据集成

Trace 录制完成后，如果启用了 CrUX（默认），系统会自动请求 Google Chrome User Experience Report API，获取真实用户的性能数据：

performance.ts — CrUX 集成 async function populateCruxData(result: TraceResult) { // 1. 获取 CrUX Manager 实例 const cruxManager = CrUXManager.instance(); // 2. 收集所有相关 URL const urls = [...result.parsedTrace.insights.values()] .map(c => c.url.toString()); urls.push(result.parsedTrace.data.Meta.mainFrameURL); // 3. 并发请求每个 URL 的 CrUX 数据 const cruxData = await Promise.all( urls.map(url => cruxManager.getFieldDataForPage(url)) ); // 4. 附加到 Trace 结果 result.parsedTrace.metadata.cruxFieldData = cruxData; }
中文解读
从 Insights 集合和主框架 URL 中收集所有需要查询 CrUX 的 URL。
并发请求每个 URL 的真实用户性能数据（LCP、FID、CLS 等）。
将 CrUX 数据附加到 Trace 结果的 metadata 中，最终格式化输出时一并展示。
这样 LLM 就能同时看到实验室数据和真实用户数据，做出更准确的分析。

内存分析：Heap Snapshot

内存工具（实验性）允许 LLM 分析 JavaScript 堆内存，找出内存泄漏：

💾
take_heapsnapshot

拍摄当前页面的 V8 堆快照，保存为文件。使用 DevTools 前端的 HeapSnapshotWorker 解析。

📊
get_heapsnapshot_summary

按构造函数分组统计对象数量和保留大小，快速定位可疑类型。

🔍
get_heapsnapshot_retainers

查看某个对象的保留链（GC Root 到该对象的路径），这是定位内存泄漏的关键。
