架构选型指南

你是技术负责人，需要决定是否采用 Hello-Agents 的方案来构建智能体。这里给你一份决策参考。

它解决什么问题？核心哲学是什么？

Hello-Agents 面向的是从零构建 AI Native Agent 这个需求——不是用 Dify/Coze 拖拽搭建流程式 Agent，而是用代码亲手实现能自主推理和行动的 AI。

问题定位 系统性、重实践的智能体构建教程极度匮乏，新手不知从何入手
核心哲学 AI Native Agent（非流程驱动）+ 从代码层面理解原理 + 亲手构建而非调 API
教学路线 16 章从 ELIZA 到赛博小镇，基础理论 → 经典范式 → 框架实战 → 综合项目
代码量 覆盖 Python + Vue + Godot 三种技术栈，含 15+ 章配套完整可运行代码

适合 vs 不适合的场景

✅ 适合

  • 想从零理解 Agent 原理的学习者
  • 需要高度定制化的 Agent 场景
  • 有 Python 基础的技术人员
  • 研究性项目或学术实验
  • 想成为 AI 工程师的转型者

❌ 不适合

  • 没有任何编程基础的人
  • 需要 5 分钟快速搭建 Agent 的场景
  • 企业级高可用生产环境（直接用框架）
  • 只想要流程编排（用 Dify/Coze 更好）
  • 需要大规模并发处理

从源码中提取的硬限制

max_steps = 5 ReAct Agent 默认最大循环步数。防止无限循环，但复杂任务可能不够用（可在初始化时修改）
max_tool_iterations = 3 SimpleAgent 的工具调用最大轮次。每次调用 LLM → 检查工具调用 → 执行 → 喂回结果，最多 3 轮
temperature = 0 默认温度为 0（最确定性的输出）。Agent 需要可预测的行为，不适合高创造性场景
TOOL_CALL 正则 工具调用解析依赖正则匹配，格式不严格就会解析失败。这是纯文本解析方案的固有局限

成本与结论

1
学习成本

约 40-60 小时通读全部 16 章 + 完成代码实践。需要 Python 基础

2
API 成本

实际运行代码需要 LLM API 费用。按 OpenAI 定价，一次完整 ReAct 循环约消耗 1000-3000 tokens

3
时间成本

从零到能构建自己的 Agent 框架，认真学完整教程约需 2-3 周

🎯
最终判定

Hello-Agents 是目前中文社区最全面的 AI Native Agent 教程。它不适合直接上生产，但如果你想真正理解 Agent 的内部工作机制，或者想亲手构建一个自己的框架，它是最系统的起点。MIT 协议开源，完全免费。
