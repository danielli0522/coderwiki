MCP 工具箱：代理怎么用图谱

九个工具，让 AI 代理从"盲目搜索"变成"精确导航"

MCP：AI 代理的"万能接口"

MCP 就像 AI 代理的 USB 接口——任何工具只要实现了 MCP 协议，AI 就能直接调用。CodeGraph 通过 MCP 暴露了 9 个工具，覆盖从快速搜索到深度追踪的各种场景。

💡
关键设计原则

工具设计的核心是"一次调用解决问题"。codegraph_context 把搜索、节点详情、调用链、源码全部打包成一次返回——AI 不需要链式调用 5-6 个工具来拼凑答案。

九把利器

MCP 工具清单 从最常用到最专业
codegraph_context 主力工具——一次返回完整上下文
codegraph_explore 多符号源码聚合——一次看多个文件
codegraph_search 快速符号搜索——只返回位置不返回代码
codegraph_node 单个符号详情——带调用轨迹的深度查看
codegraph_trace 调用路径追踪——从 A 到 B 怎么走的
codegraph_callers / callees 谁调了它 / 它调了谁
codegraph_impact 影响半径分析——改了这个会波及谁
codegraph_status 索引状态查看
codegraph_files 项目文件结构——比 glob 更快

工具是怎么定义的？

每个 MCP 工具就是一个 JSON 描述——告诉 AI 这个工具叫什么、接受什么参数、做什么用：

CODE 

{
  name: 'codegraph_context',
  description: 'PRIMARY TOOL - call this FIRST',
  inputSchema: {
    properties: {
      task: {
        description: 'Description of the task'
      },
      maxNodes: { default: 20 },
      includeCode: { default: true }
    }
  }
}
大白话

工具名称——AI 通过这个名字调用它。

工具说明——告诉 AI 什么时候该用这个工具。这里写"主力工具，优先调用"。

输入参数的定义。

task 参数——用自然语言描述你想了解什么。

最多返回多少个符号节点，默认 20 个。

是否包含源码片段，默认包含。

参数定义结束。

整个工具定义结束。

⚠
安全硬限制

所有工具输出上限 15000 字符（MAX_OUTPUT_LENGTH），输入上限 10000 字符（MAX_INPUT_LENGTH），路径上限 4096 字符。这些是代码里的常量，防止恶意或失控的请求吃光资源。
