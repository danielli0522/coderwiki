回顾 CodeGraph 核心知识，通过实践巩固技能，探索代码图谱生态

核心知识回顾

🗺

代码地图概念

AI 代理面对大代码库时反复 grep/read 浪费大量 token。CodeGraph 预建本地知识图谱，让 AI 像查地图一样查代码结构，平均省 35% 成本、减 57% token、快 46%。

🏗

三层架构

六层管道分工协作：Extraction（解析源码）-> Resolution（关联引用）-> Database（SQLite 存储）-> Graph（遍历查询）-> Context（整理输出）-> MCP（对外接口）。每层职责单一，可独立测试。

⛳

索引管道

四道工序：tree-sitter 解析 AST 提取符号 -> ReferenceResolver 匹配未关联引用 -> 写入 SQLite（FTS5 + WAL）-> FileWatcher 增量同步。支持 18+ 语言，还能识别 Express/React 等框架模式。

🔧

MCP 工具集

9 个 MCP 工具覆盖从搜索到追踪的全部场景。主力工具 codegraph_context 一次返回完整上下文（符号+调用链+源码）。智能预算控制根据项目规模自动调整输出量。

⚖

技术选型

CodeGraph vs RAG/Grep/全量上下文：本地 AST 解析、确定性提取、零 API 成本、100% 隐私。适合 500+ 文件的中大型项目。MIT 许可证，一行命令安装卸载。

最佳实践清单

📄 CodeGraph 使用最佳实践
📦 索引策略
✓ 首次 clone 后立即 codegraph init + index，让索引先跑起来
✓ 保持 FileWatcher 运行，代码变化时自动增量同步
✗ 不要频繁手动 codegraph index，增量同步已经够用
✗ 不要把 .codegraph/ 目录加入 git（本地生成，无需共享）
🔎 查询技巧
✓ 优先用 codegraph_context，它是"一次调用解决问题"的主力工具
✓ 用自然语言描述任务意图，比精确符号名更有效
✓ 追踪调用路径时用 codegraph_trace，分析影响范围用 codegraph_impact
✗ 不要链式调用 5-6 个工具拼凑答案，context 工具已内置搜索+追踪
⚡ 性能优化
✓ 大项目（5000+ 文件）用 maxNodes 控制返回节点数，避免上下文爆炸
✓ 只需要位置信息时用 codegraph_search，不需要源码时设 includeCode=false
✓ trace 工具限制最大 7 跳（MAX_HOPS），超过说明查询本身需要细化
✗ 不要一次性查询整个模块的所有符号，分而治之更高效

下一步学习方向

🤖

Understand-Anything

同样基于 tree-sitter 的代码理解框架，但侧重"一键理解任意开源项目"。结合 AI 摘要和交互式探索，适合快速上手陌生代码库。

📈

GitNexus

把 Git 提交历史转化为知识图谱——谁改了什么、变更的影响路径、代码热力图。将代码结构分析与版本历史结合，做"时间维度"的代码理解。

🌍

Graphiti

将对话记忆构建为知识图谱的方案。如果把 CodeGraph 理解为"静态代码图谱"，Graphiti 就是"动态记忆图谱"——两者结合可以实现"既懂代码又记得对话"的 AI 代理。
