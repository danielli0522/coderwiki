从源码到知识图谱：索引管道

一个文件变成图谱数据，要经历四道工序

代码索引就像石油炼化

原油（源文件）进厂后，先蒸馏（解析语法树），再裂化（提取符号），然后调和（关联引用），最后入库（写入数据库）。每一道工序都依赖上一道的结果。

四道工序详解

1
ExtractionOrchestrator — 用 Tree-sitter 解析语法树

读取每个源文件，用 tree-sitter 构建 AST，从中提取函数、类、方法等符号作为节点（Node），同时提取包含、导入等初步关系作为边（Edge）。支持 18+ 种语言。

2
ReferenceResolver — 解析未关联的引用

提取阶段留下了一批"我知道这个名字但不知道它指向谁"的引用。Resolver 通过三种策略匹配：框架模式识别（Express/Laravel 路由）、import 路径追踪、名称模糊匹配。还能识别 React 组件渲染、Vue 模板事件等动态分派。

3
Database — 写入 SQLite

所有节点和边存入本地 SQLite 文件（.codegraph/ 目录下），用 FTS5 建全文搜索索引。用 WAL 模式实现并发安全。

4
FileSync — 监控变化自动更新

启动 FileWatcher 监控文件变化（macOS 用 FSEvents，Linux 用 inotify），文件一改就触发增量同步——只重新解析变化的文件，不重建整个图谱。

代码长什么样？

下面是 CodeGraph 主类中索引流程的核心代码：

CODE 

async indexAll(options?: IndexOptions) {
  return this.indexMutex.withLock(async () => {
    this.fileLock.acquire();
    const result = await this.orchestrator
      .indexAll(options.onProgress);
    if (result.filesIndexed > 0) {
      await this.resolveReferencesBatched();
    }
    this.db.runMaintenance();
    return result;
  });
}
大白话

indexAll 是"全部重新索引"的入口。

先加一把进程内的互斥锁，防止两个索引任务同时跑。

再获取一个跨进程的文件锁，防止 CLI 和 MCP 同时写数据库。

让 Orchestrator 去解析所有文件，提取符号和初步关系。

如果确实有新文件被索引了……

……就启动引用解析，把未关联的引用匹配到实际符号。

最后做一些数据库维护（刷新统计信息、压缩日志）。

返回索引结果（多少文件、多少节点、多少边）。

锁在函数结束时自动释放。

不只是语法——还有框架智慧

CodeGraph 不只看语法结构，还能识别 Web 框架 的路由模式。比如它知道 Express 的 app.get('/users', handler) 意味着有一个路由端点。

⚙

Express / NestJS

识别 app.get/post/put/delete 路由注册，建立 route 节点和 handler 引用边。

🚀

React / Svelte / Vue

识别组件渲染关系、JSX 子组件引用、Vue 模板事件绑定等动态分派。

☕

Spring / Laravel / Django

识别注解路由（@GetMapping）、控制器映射、Django URL 配置等。

📦

Cargo Workspace

解析 Rust workspace 成员和模块依赖关系，跨 crate 引用追踪。
