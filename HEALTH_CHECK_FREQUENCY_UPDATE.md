# 健康检查频率更新

## 更新概述

根据性能优化需求，将健康检查频率从高频调整为低频，以减少服务器负载并提高系统性能。

## 修改内容

### 1. 系统状态检查

- **文件**: `frontend/static/js/components/system_status.js`
- **原频率**: 30 秒
- **新频率**: 3 分钟 (180000 毫秒)
- **影响**: 减少系统状态 API 调用频率

### 2. 任务监控

- **文件**: `frontend/static/js/task_progress.js`
- **原频率**: 3 秒
- **新频率**: 3 分钟 (180000 毫秒)
- **影响**: 减少任务状态查询频率

### 3. 任务进度组件

- **文件**: `frontend/static/js/components/task_progress.js`
- **原频率**: 5 秒
- **新频率**: 3 分钟 (180000 毫秒)
- **影响**: 减少任务列表刷新频率

### 4. 任务详情页面

- **文件**: `frontend/static/js/task_detail.js`
- **原频率**: 3 秒
- **新频率**: 3 分钟 (180000 毫秒)
- **影响**: 减少任务详情更新频率

### 5. 任务管理页面

- **文件**: `frontend/static/js/task.js`
- **原频率**: 5 秒
- **新频率**: 3 分钟 (180000 毫秒)
- **影响**: 减少任务列表轮询频率

### 6. 设置界面更新

- **文件**: `frontend/templates/task/progress.html`
- **更新**: 添加 3 分钟选项作为默认选择
- **影响**: 用户界面支持新的刷新间隔

## 性能影响

### 正面影响

1. **减少服务器负载**: API 调用频率降低约 90%
2. **降低网络流量**: 减少不必要的 HTTP 请求
3. **提高响应速度**: 服务器资源更多用于处理实际业务请求
4. **减少数据库查询**: 降低数据库连接池压力

### 潜在影响

1. **实时性降低**: 状态更新延迟增加
2. **用户体验**: 任务状态变化反馈稍慢

## 配置选项

用户仍可通过设置界面调整刷新频率：

- 1 秒 (1000ms)
- 3 秒 (3000ms)
- 5 秒 (5000ms)
- 10 秒 (10000ms)
- 30 秒 (30000ms)
- **3 分钟 (180000ms)** - 默认选择

## 建议使用场景

### 生产环境

- **推荐**: 3 分钟间隔
- **原因**: 平衡性能和实时性需求

### 开发环境

- **推荐**: 30 秒间隔
- **原因**: 便于调试和快速反馈

### 高负载环境

- **推荐**: 5 分钟或更长间隔
- **原因**: 最大化性能优化效果

## 监控建议

1. **观察系统性能**: 监控 CPU、内存使用率变化
2. **用户反馈**: 收集用户对响应速度的反馈
3. **错误率**: 监控 API 错误率是否降低
4. **网络流量**: 观察网络请求量减少情况

## 回滚方案

如需恢复高频检查，可以：

1. 将默认值改回原来的设置
2. 或者通过环境变量控制刷新频率
3. 提供用户可配置的刷新间隔选项

## 后续优化

2. **WebSocket**: 考虑使用 WebSocket 替代轮询
3. **缓存策略**: 实现客户端缓存减少重复请求
4. **批量更新**: 合并多个状态更新请求
